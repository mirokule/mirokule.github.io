分类

  • 内存占用
    In-place Sort（不占用额外内存或占用常数的内存）：插入排序、选择排序、冒泡排序、堆排序、快速排序。
    Out-place Sort：归并排序、计数排序、基数排序、桶排序。

  • 稳定性
    Stable Sort：插入排序、冒泡排序、归并排序、计数排序、基数排序、桶排序。
    Unstable Sort：选择排序(5 8 5 2 9)、快速排序、堆排序。

  • 复杂度
    O(n2)：插入排序、选择排序、冒泡排序
    nlogN：快速排序、归并排序、堆排序

插入排序 Insertion Sort

思路：集合A初始为空，每次选择1个元素插入集合A，直到所有元素都插入。
实现：

for i=2 to n
    j = i - 1
    key = A[i]
    while j>0 && A[j]>key
        A[j+1] = A[j]
        j--
    A[j+1] = key

冒泡排序 Bubble Sort

思路：两两交换，最大或最小元素将在首次冒出
实现：

for i=1 to n
    for j = n to i+1
        if A[j]<A[j-1]
            swap(A[j], A[j-1])

选择排序 Selection Sort

思路：每次找剩余元素中的最小值
实现：

for i=1 to n-1
    minIndex = i
    for j = i+1 to n
        if A[minIndex]>A[j]
            minIndex = j
    swap(A[minIndex], A[i])

归并排序 Merge Sort

思路：使用分治法，将若干个子序列分别排序，然后合并
实现：

Merge_sort(A) {
    recursive_mergesort(A, 1, length(A))
}
recursive_mergesort(A,p,q) {
    if p<q
        m = (p+q)/2
        recursive_mergesort(A,p,m)
        recursive_mergesort(A,m+1,q)
        merge(A,p,m,q)
}
merge(A,p,m,q) {
    a = m-p+1 //子序列长度, 例p=1, m=3, q=5
    b = q-m
    new Array L[a+1] & R[b+1]
    for i=1 to a
        L[i] = A[p+i-1]
    for i=1 to b
        R[i] = A[m+i]
    L[a+1] = INFINITY
    R[b+1] = INFINITY
    i = j = 1
    for k = p to q
        if L[i] < R[j]
            A[k] = L[i]
            i++
        else if L[i] > R[j]
            A[k] = R[j]
            j++
}

讨论
问：归并排序的缺点是什么？
答：他是Out-place sort，因此相比快排，需要很多额外的空间。

问：为什么归并排序比快速排序慢？
答：虽然渐近复杂度一样，但是归并排序的系数比快排大。

问：对于归并排序有什么改进？
答：就是在数组长度为k时，用插入排序，因为插入排序适合对小数组排序。在算法导论思考题2-1中介绍了。复杂度为O(nk+nlg(n/k)) ，当k=O(lgn)时，复杂度为O(nlgn)

快速排序 Quick Sort

思想：也是基于分治法，随机选取元素k，将大于k和小于k的元素分成子序列，继续对子序列选取元素并分裂。
实现：

quick_sort(A) {
    recursive_quicksort(A, 1, length(A))
}
recursive_quicksort(A, p, q) {
    if p<q
        r = partion(A, p ,q)
        recursive_quicksort(A, p, r-1)
        recursive_quicksort(A, r+1, q)
}
partition(A,p,q) {
    i = p-1
    pivot = A[q]
    for j=p to q-1 //遍历序列，每遇到一个比pivot小的，都换到前面去
        if A[j] <= pivot
            i++
            swap(A[i], A[j])
    swap(A[i+1], A[q])
    return i+1
}

堆排序 Heap Sort

思想：运用了最小堆、最大堆，依次输出堆顶元素
实现：

heapsort(A) {
    build_max_heap(A)
    for i=n to 2
        swap(A[1], A[heapsize])
        heapsize--
        MAX_HEAPIFY(A,1)
}
build_max_heap(A){
    for i=floor(n/2) to 1
        MAX_HEAPIFY(A,i)
}
MAX_HEAPIFY(A,i) {
    heapsize = heapsize[A]
    largest = i
    if left(i)<=heapsize && A[largest]<A[left(i)]
        largest = left(i)
    else if right(i)<=heapsize && A[largest]<A[right(i)]
        largest = right(i)
    if(largest!=i)
        swap(A[i],A[largest])
        MAX_HEAPIFY(A,largest)
}

计数排序 Counting Sort

思想：对于给定的输入序列中的每一个元素x，确定该序列中值小于x的元素的个数（此处并非比较各元素的大小，而是通过对元素值的计数和计数值的累加来确定）。
一旦有了这个信息，就可以将x直接存放到最终的输出序列的正确位置上。
条件：元素的取值范围有一定限制，即k=O(n)。
实现：

n = length[A]
new array B[n] & C[k+1]
for i=0 to k
    C[i] = 0
for i=1 to n 
    C[A[i]] ++  //A[i]取值范围有限制
for i=1 to k
    C[i] = C[i] + C[i-1]
for i=n to 1
    B[C[A[i]]] = A[i]
    C[A[i]]--

桶排序 Bucket Sort

思想：将数组分到有限数量的桶里，各桶分别排序后合并
实现：

n = length[A]
new buckets B[n]
for i=1 to n
    insert A[i] to B[nA[i]]
for i=0 to n-1
    sort B[i]
concatenate B[]

基数排序 Radix Sort

思想：每次按1个数位排序，依次实现由低位到高位的有序
实现：

for i = 1 to d
