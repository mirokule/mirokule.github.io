Boosted Tree

CART

Boosted Tree最基本的组成部分是回归树(regression tree)，也称作CART。
CART会把输入根据输入的属性分配到各个叶子节点，每个叶子节点上会对应一个实数分数。

Tree Ensemble

1. 模型和参数

一棵树往往过于简单，将多棵树的组合称作tree ensemble。
在这个模型中，参数是树的结构，以及每个叶子节点上的预测分数。
每个样本的最终预测结果是每棵树预测分数的和。

模型可以写作：
$$ \hat{y_i} = \sum_{k=1}^Kf_k(x_i)\;\;\; f_k \in \boldsymbol{F} $$
其中每个$f$是一个在函数空间$\boldsymbol{F}$里面的函数，$\boldsymbol{F}$对应所有regression tree的集合。

2. 目标函数

$$ Obj(\Theta) = \sum_i^nl(y_i, \hat{y_i}) + \sum_{k=1}^K\Omega(f_k)$$
第一部分损失是训练误差，如MSE、logistic loss等；
第二部分正则是每棵树的复杂度的和，复杂度有多种定义方式，比如叶节点个数+叶节点分数平方和。

3. 优化算法

参数可以认为是在函数空间里面，不能采用传统的如SGD之类的算法，采用additive training的方式。
每次保留原来的模型不变，加入一个新的$f$，即一棵新树。
$$ \hat{y_i}^{(t)} = \sum_{k=1}^tf_k(x_i) = \hat{y_i}^{(t-1)} + f_t{x_i} $$
选取新树$f$的标准是，使目标函数$Obj(\Theta)$尽量最大地降低。

sklearn-Pipeline

发表于 2018-03-28
字数统计: 344

Pipeline将若干estimators按先后顺序组合到一起，实现了对全部步骤的流式化封装。
最后一个estimator可以是任何类型，前面的N-1个estimator必须是transformer(有transform方法)。
Pipeline在机器学习算法中得以应用的根源在于，参数集在新数据集(比如测试集)上的重复使用。

1. pipeline.Pipeline

class sklearn.pipeline.Pipeline(steps, memory=None)
Pipeline用一系列(key, value)对来构造。

参数

  • steps: list of tuples that are chained
  • memory: cache文件路径，避免重复计算

属性

  • named_steps: Read-only attribute to access any step parameter

方法

  • fit(X, y=None, **fit_params): fit and transform all the transforms, fit the final estimator
  • fit_transform(X, y=None, **fit_params): fit and transform with the final estimator
  • fit_predict(X, y=None, **fit_params): apply fit_predict of the last step
  • predict(X): apply transforms, and predict with the final estimator
  • predict_proba(X): apply transforms, and predict_proba of the final estimator
  • predict_log_proba(X): apply transforms, and predict_log_proba of the final estimator
  • score(X, y=None, sample_weight=None): apply transforms, and score with the final estimator
  • decision_function(X): apply transforms, and decision_function of the final estimator
  • get_params
  • set_params

实例

2. pipeline.make_pipeline

sklearn.pipeline.make_pipeline(*steps, **kwags)
make_pipeline是构造pipeline的便捷写法，不允许命名，各模块将自动命名。

参数

  • *steps: list of estimators
  • memory: 同上

返回

  • p: Pipeline

1.KFold

class sklearn.model_selection.KFold(n_splits=3, shuffle=False, random_state=None)
将所有数据划成k份，使用k-1份训练，保留1份用于验证。
通常建议使用5份或10份；当k=n时，等同于Leave One Out。

参数

  • n_splits: number of folds，最小2
  • shuffle: 划分前是否Shuffle
  • random_state: 当shuffle==True时使用的种子

方法

  • get_n_splits(X=None, y=None, groups=None): number of splits
  • split(X, y=None, groups=None): generate indices to split data into training and test set

实例

  • n_repeats: number of times repeated

  • get_n_splits(X=None, y=None, groups=None)

实例

3.ShuffleSplit

class sklearn.model_selection.ShuffleSplit(n_splits=10, test_size=’default’, train_size=None, random_state=None)
可以分别指定splits和test_size，样本shuffle后再划分。
和KFold相比，重复次数和划分方式实现了解耦。

参数

  • n_splits: number of re-shuffling & splitting
  • test_size: 如果train_size没有指定，则默认0.1
  • train_size: (0.0~1.0)，或者用整数表示的样本数
  • random_state:

方法

  • get_n_splits(X=None, y=None, groups=None)
  • split(X, y=None, groups=None)

实例

4.Leave One Out(LOO)

每次使用n-1个样本训练模型，用剩余的1个样本验证。
每次训练出的模型差别不大，测试结果的波动较大。
通常不如划分为5份或10份。

5.Leave P Out(LPO)

每次保留p个样本用于验证，使用n-p个样本训练。
与LOO和KFold的区别在于，p>1时测试集可能是重复的。

scipy-BoxCox

发表于 2018-03-25
字数统计: 305

Box-Cox变换

使用Box-Cox变换一般都可以成功地数据正态变换，但在二分变量或较少水平的等级变量的情况下，变换不能成功。
变换后，残差可以更好地满足正态性、独立性等假设前提，降低伪回归的概率。

$$ y = \frac{x^\lambda - 1}{\lambda}\;\;for\;\;\lambda > 0$$
$$ y = Log(x) \;\; for \;\; \lambda = 0 $$

相关函数

1.scipy.stats.boxcox(x, lmbda=None, alpha=None)

参数

  • x: 输入序列，要求一维、正数
  • lmbda: 若为None，则自动寻找maximizes the log-likelihood function的参数$\lambda$
  • alpha: 若不为None，则计算100*(1-alpha)%置信区间

返回

  • boxcox: 转换后的ndarray
  • maxlog: 输入lmbda为None时的自适应参数
  • (min_ci, max_ci): 给定alpha的置信区间
2.scipy.special.boxcox(x, lmbda)
  • 若$x<0$则返回nan.
  • 若$x==0,\lambda<0$返回-inf

$$ y = \frac{x^\lambda - 1}{\lambda}\;\;for\;\;\lambda != 0$$
$$ y = Log(x) \;\; for \;\; \lambda = 0 $$

参数

  • x: array_like
  • lmbda: 参数

返回

  • y: array
3.scipy.special.boxcox1p(x, lambda)

返回(1+x)的box-cox变换，通过平移解决$x=0$的边界情况

  • 若$x<-1$则返回nan
  • 若$x==-1, \lambda<0$则返回-inf

$$ y = \frac{(1+x)^\lambda - 1}{\lambda}\;\; for \;\; \lambda != 0 $$
$$ y = Log(1+x)\;\; for \;\; \lambda = 0 $$

参数

  • x: array-like
  • lmbda: 参数

返回

  • y: array
