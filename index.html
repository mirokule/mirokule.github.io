Keras序贯模型Sequential

发表于 2018-05-05
字数统计: 953

序贯模型是多个网络层的线性堆叠。

快速开始

构造
可以通过传递一个layer的list来构造模型，也可以通过add()方法将Layer逐个加入。

输入
Sequential的第一层需要接受一个输入数据shape的参数，后面的各个层可以自动推导出中间数据的shape.
为第一层指定输入数据shape的方法包括：

  • input_shape: tuple类型，不包括batch大小
  • input_dim: 某些层支持该参数
  • batch_size: 某些层需要指定固定大小的batch_size

编译
训练模型之前，需要通过compile对模型编译。
compile(self, optimizer, loss, metrics=None, sample_weight_mode=None)

  • optimizer: 优化器，必须
  • loss: 目标函数，必须
  • metrics: 评价指标，可选
  • sample_weight_mode: 默认”None”按样本赋予权值；”temporal”按时间步赋予权值

训练
fit(self, x, y, batch_size=32, epochs=10, verbose=1, callbacks=None,
validation_split=0.0, validation_data=None, shuffle=True,
class_weight=None, sample_weight=None, initial_epoch=0)

  • x: 单个输入numpy array，多个输入是list of numpy array
  • y: numpy array
  • batch_size: 优化一步的样本数
  • epochs: 训练的总轮数
  • verbose: 日志显示，0不输出，1输出进度条，2每个epoch输出一行记录
  • callbacks: list，其中的元素是Callback函数
  • validation_split: float(0~1)，验证集的比例
  • validation_data: 指定验证集(X,y)，将覆盖validation_split
  • shuffle: 是否随机打乱顺序
  • class_weight: 字典，不同类别映射为不同的权重
  • sample_weight: 样本权重，numpy array
  • initial_epoch: 从指定的epoch开始训练，用于继续之前的训练过程

其他方法

  • add(self, layer): 向模型中添加一个层
  • pop(self): 弹出模型最后的一层，无返回值
  • evaluate(self, x, y, batch_size=32, verbose=1, sample_weight=None): 计算误差
  • predict(self, x, batch_size=32, verbose=0): 预测
  • train_on_batch(self, x, y, class_weight=None, sample_weight=None): 根据一个batch更新参数
  • test_on_batch(self, x, y, sample_weight=None): 在一个batch上评估模型
  • predict_on_batch(self, x): 在一个batch上测试
  • fit_generator(self, generator, steps_per_epoch, epochs=1, verbose=1, callbacks=None, validation_data=None, validation_steps=None, class_weight=None, max_q_size=10, workers=1, pickle_safe=False, initial_epoch=0)
  • evaluate_generator(self, generator, steps, max_q_size=10, workers=1, pickle_safe=False)
  • predict_generator(self, generator, steps, max_q_size=10, workers=1, pickle_safe=False, verbose=0)

Keras模型Models

发表于 2018-05-05
字数统计: 254

Keras有两种类型的模型，序贯模型(Sequential)和函数式模型(Functional)。
函数式模型应用更广泛，序贯模型是函数式模型的一种特殊情况。

两类模型有一些通用方法：

  • model.summary(): 输出模型概况，实际调用的是keras.utils.print_summary
  • model.get_config(): 返回包含配置信息的python字典，配置信息可用于重构
  • model.get_layer(): 根据层名称或下标获得层对象
  • model.get_weights(): 返回模型权重，类型为numpy array
  • model.set_weights(): 从numpy array将权重载入给模型
  • model.to_json: 返回代表模型的JSON字符串，只包含网络结构，不包括权值
  • model.to_yaml: 返回代表模型的YAML字符串
  • model.save_weights(filepath): 将权重保存到指定路径，文件类型HDF5(.h5)
  • model.load_weights(filepath, by_name=False): 从HDF5中加载权重

Keras噪声层Noise

发表于 2018-05-05
字数统计: 170

噪声层用于给数据附加随机的噪声，用以克服过拟合。
也是一种正则化，仅在训练时使用。

GaussianNoise

添加均值为0，标准差为stddev的高斯噪声。

GaussianDropout

乘以将均值1，标准差为$\sqrt{\frac{rate}{1-rate}}$ 的噪声。
multiplicative noise 指在信号处理中，乘进原信号的随机噪声。

AlphaDropout

使输入数据的均值和方差保持不变，保持数据的自规范性。
通过随机将激活值设为负饱和值，alpha dropout 和selu激活函数配合较好。
噪声的标准差也是$\sqrt{\frac{rate}{1-rate}}$

Keras规范化层BatchNormalization

发表于 2018-05-04
字数统计: 240

在每次迭代的batch里，对上一层的输出规范化。算法分两步：

  1. Normalize: 将输出变为均值0，标准差1；
  2. Scale and Shift: 加入新的均值和方差，避免失去学习到的信息

用公式表示算法流程：
$$\begin{align}
\mu_B &\leftarrow \frac{1}{m} \sum_{i=1}^m x_i &&\text{batch mean} \\
\sigma_B^2 &\leftarrow \frac{1}{m} \sum_{i=1}^m \;(x_i-\mu_B)^2 &&\text{batch variance} \\
\hat{x_i} &\leftarrow \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}} &&\text{normalize} \\
y_i &\leftarrow \gamma\hat{x_i} + \beta \equiv BN_{\gamma,\beta}\;(x_i) &&\text{scale and shift}
\end{align} $$

代码

  • axis: 指定要规范化的轴
  • momentum: 移动平均和移动方差的冲量，公式如下：
    moving_mean = momentum × moving_mean + (1 - momentum) × batch_mean
    moving_var = momentum × moving_var + (1 - momentum) × batch_var
  • epsilon: 加到方差里的小浮点数，避免除数为0
  • center: 若True，则给规范化后的tensor加上偏移量beta
  • scale: 若True，则乘以gamma

Keras嵌入层Embedding

发表于 2018-05-04
字数统计: 137

Embedding嵌入层将正整数转换为固定大小的向量，例如将字符的数字编码转为word embedding向量([3790]->[3,5,0,12])
Embedding层只能作为模型的第一层。

输入shape: (batch_size, sequence_length)
输出shape: (batch_size, sequence_length, output_dim)

  • input_dim: 输入字典长度，最大下标+1
  • output_dim: 输出长度
  • mask_zero: 是否将输入中的’0’当做padding而全部忽略
  • input_length: 输入序列的长度；不指定时后续层的输出维度无法推断

Keras循环层Recurrent

发表于 2018-05-04
字数统计: 714

Recurrent

循环层的基类，所有的循环层(LSTM, GRU, SimpleRNN)都继承本层。

  • cell: RNN cell实例，包括
    • call(input_at_t, states_at_t)方法，返回(output_at_t, states_at_t_plus_1)
    • state_size
  • return_sequences: 默认False返回最后一个输出；若True则返回每个时刻的输出序列
  • return_state: 是否返回最后时刻的隐状态
  • go_backwards: 是否将输入序列逆向处理，即将最后一个当做t=1
  • stateful: 当前batch下标为i的最终状态，将会被用作下个batch下标为i的初始状态
  • unroll: 若True则循环层将被展开，加速计算，但会占用更多内存

SimpleRNN, SimpleRNNCell

一种全连接RNN，输出会回馈到输入。

  • units: 输出维度

GRU, GRUCell

Gated Recurrent Unit有两个变种。
默认实现(1406.1078v3)，reset gate 在矩阵相乘前作用于hidden state；
另一种实现(1406.1078v1)的顺序则相反，和CuDNNGRU (GPU-only) 兼容。
调用时使用’reset_after’=True，’recurrent_activation=’sigmoid’.

LSTM

Long Short Term Memory Layer.

  • unit_forget_bias: 若为true将会在forget gate初始化时加1，并强制 bias_initializer=”zeros”
  • implementatioin: mode 1会使用大量小型计算，mode 2会使用少量大型计算；根据设备与任务选择

ConvLSTM2D

和LSTM类似，但是输入变换和循环变换都是通过卷积完成的。

CuDNNGRU, CuDNNLSTM

Fast GRU/LSTM implementation backed by CuDNN.
只能在GPU上运行，以TensorFlow作为后端。
