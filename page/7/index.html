Box-Cox变换

使用Box-Cox变换一般都可以成功地数据正态变换，但在二分变量或较少水平的等级变量的情况下，变换不能成功。
变换后，残差可以更好地满足正态性、独立性等假设前提，降低伪回归的概率。

$$ y = \frac{x^\lambda - 1}{\lambda}\;\;for\;\;\lambda > 0$$
$$ y = Log(x) \;\; for \;\; \lambda = 0 $$

相关函数

1.scipy.stats.boxcox(x, lmbda=None, alpha=None)

参数

  • x: 输入序列，要求一维、正数
  • lmbda: 若为None，则自动寻找maximizes the log-likelihood function的参数$\lambda$
  • alpha: 若不为None，则计算100*(1-alpha)%置信区间

返回

  • boxcox: 转换后的ndarray
  • maxlog: 输入lmbda为None时的自适应参数
  • (min_ci, max_ci): 给定alpha的置信区间
2.scipy.special.boxcox(x, lmbda)
  • 若$x<0$则返回nan.
  • 若$x==0,\lambda<0$返回-inf

$$ y = \frac{x^\lambda - 1}{\lambda}\;\;for\;\;\lambda != 0$$
$$ y = Log(x) \;\; for \;\; \lambda = 0 $$

参数

  • x: array_like
  • lmbda: 参数

返回

  • y: array
3.scipy.special.boxcox1p(x, lambda)

返回(1+x)的box-cox变换，通过平移解决$x=0$的边界情况

  • 若$x<-1$则返回nan
  • 若$x==-1, \lambda<0$则返回-inf

$$ y = \frac{(1+x)^\lambda - 1}{\lambda}\;\; for \;\; \lambda != 0 $$
$$ y = Log(1+x)\;\; for \;\; \lambda = 0 $$

参数

  • x: array-like
  • lmbda: 参数

返回

  • y: array

Q-Q图

发表于 2018-03-21
字数统计: 223

用途

Q-Q图将两个概率分布的分位数放在一起比较，若分布线性相关，则应绘制一条直线。
固定一种为正态分布，则可观察另一套样本是否近似服从正态分布。

作图步骤

  1. 将原始观察数据从小到大排列：$x_1,x_2,…,x_n$
  2. 计算$x$对应的分位频率$p_i=(i-\frac{1}{2})/n,i=1,2,…,n$，这里$P$可以有多种定义。
  3. 计算$p$对应的标准正态分位数$q$:
    $$ P\left{X\leq q_1\right} = \int_{-\infty}^{q_1}\frac{1}{\sqrt{2\pi}}e^{-\frac{x^2}{2}} dx = p_i $$
  4. 将数对$(q_i, x_i)$画在坐标平面上。

因为$q_i$由$p_i$唯一确定，所以若数据$x_i$的分布与正态分布非常接近，则点$(q_i,x_i)$应大致成一条直线。

对称密码

对称密码(symmetric cryptography)是指在加密和解密时使用同一种密钥的方式。

1.一次性密码本
随机生成与明文等长的比特序列key，将明文msg与key做异或加密，再做异或解密。
缺点：需要保密配送随机生成的key

2.DES(Data Encryption Standard)
密钥长度56位，每次加密64位信息
3DES：用3个密钥，进行加密->解密->加密过程

3.AES(Rijndael)
由多个轮组成，分组长度128位。

非对称密码

非对称密码(asymmetric cryptography)是指在加密和解密时使用不同密钥的方式，比如公钥密码(public-key cryptography)。
加密使用公开密码，解密使用私有密码，私有密码不需要传输，解决了对称加密中的密钥配送问题。
缺点：处理速度只有对称密码的几百分之一，不适合用来对很长的消息内容加密。

1.RSA
$$ 密文 = (明文)^E\; mod\; N\;\;\;\;\; 明文 = (密文)^D\; mod\; N$$
E和N组成了公钥，用于加密；D和N组成私钥，用于解密。
计算过程：

  • 准备两个很大的质数p和q，相乘得到N
  • 求p-1和q-1的最小公倍数，得到L
  • 找到一个比L小，且和L互质的整数E
  • 找到一个比L小，且(E*D)和L互质的整数D

单向散列函数Hash

根据任意长度的消息计算出固定长度的散列值，特点：

  • 单向性：不能逆推出原信息
  • 抗碰撞(collision)：对同一输出，难以找到两个不同的输入
  • 计算速度快

实例：

  • MD4, MD5
  • SHA-1, SHA-256, SHA-384, SHA-512
  • RIPEMD-160
  • AHS, SHA-3

能识别出文件是否被“篡改”，不能识别出发送者的身份是否“伪装”

消息认证码MAC

消息认证码(message authentication code)是一种确认完整性并进行认证的技术。
输入包括任意长度的消息和双方共享的密钥，输出是固定长度的数据，称为MAC值。

数字签名

同样使用一对钥匙，不过和前文相反，私钥用来加密生成签名，公钥用来解密验证签名。

  • Standardization: 标准化，根据中心极限定理，将数据转变为近似服从标准正态分布N(0,1)
  • Normalization: 规范化，将数据映射到相同的范围，最常见的是[0,1]，此时也叫归一化，消除量纲影响，对GDA友好
  • Regularization: 正则化，在模型的cost function里增加参数惩罚项，避免过拟合

1. StandardScaler, Scale

标准化，将特征减去均值，再除以标准差，从而服从标准正态分布N(0,1)
默认列向量，每行是一次采样。

参数：

  • copy: 为True则返回拷贝，为False则修改原数据，不一定总是有效
  • with_mean: 为True则在缩放前先减去均值
  • with_std: 为True则缩放到单位方差
  • axis: 默认为0缩放列向量(每个特征)，为1则缩放行向量(每次采样)

属性：

  • scale_:
  • mean_: 特征原均值
  • var_: 特征原方差
  • n_samples_seen_: 处理的样本数

方法：

  • fit(X, y=None): 计算均值和标准差
  • transform(X, copy=None): 变换数据
  • fit_transform(X, y=None, **fit_params): 计算，并变换数据
  • inverse_transform(X, copy=None): 逆向变换，得到原数据
  • partial_fit(X, y=None): 在线计算均值和标准差
  • get_params(deep=True): 返回参数
  • set_params(**params): 设定参数

2. Normalizer, Normalize

默认列向量，每行是一次采样。
将每行(所有变量的一次采样)缩放到单位长度，l1是曼哈顿距离，l2是欧几里德距离。
比如某行为(a,b,c)，l2 norm，则缩放后$a^2 + b^2 + c^2 = 1$

参数：

  • norm: 可以选’l1’, ‘l2’, ‘max’
  • copy: 为False则修改原数据，为True则返回拷贝
  • axis: 为0则缩放列向量(每个特征), 默认为1缩放行向量(每次采样)

方法：

  • fit(X, y=None): 无作用，只是为了实现接口
  • transform(X, copy=None): 缩放每个非零行
  • fit_transform(X, y=None, **fit_params): 带参数缩放
  • set_params(**params): 设定参数
  • get_params(deep=True): 返回参数
