神经网络配置与性能提升

发表于 2018-05-28
字数统计: 486

激活函数Activation

  1. Sigmoid: 现已基本不用，缺点有三

    • 饱和区会杀死梯度
    • 输出0~1全为正，那么梯度全为正负，使w更新方向受限制(所以输入数据mean=0最好)
    • 指数运算开销大

  2. Tanh: 解决了Mean=0的问题，仍会杀死梯度、指数运算

  3. Relu: 收敛速度约6倍，没有mean=0，会有神经元“死亡”；一般为首选
  4. Leaky Relu、Max Out、ELU: 可尝试

初始化Initialization

  1. 全为0，所有节点结果相同；太小，梯度消失；太大，网络饱和
  2. Xavier/Glorot: 使输入输出方差相同，适合搭配tanh
  3. He: 方差除2，适合搭配relu

优化器Optimizer

  1. SGD: 经典方法，通常较慢
  2. 动量: 收敛速度加快，但容易over shooting，即步长降低得慢
  3. RMSProp: 各方向步长尽量均衡
  4. Adam: RMSProp + 动量，一般为首选

损失函数Loss

  1. 铰链Hinge: 主要用于SVM
  2. 互熵Cross Entropy Loss: 用于Logistic、Softmax分类
  3. 平方Square Loss: 用于最小二乘法(OLS)
  4. 指数Exponential Loss: 用于Adaboost等集成学习算法

克服overfitting

  1. Batch Normalization一般为首选，效果不好时再考虑其他手段；通常在Activation层之前。
  2. Dropout相当于内部集成，注意测试时不用drop，得分要乘以比例p
  3. Data Augmentation: 生成更多训练数据

Q: 如何识别？
A: 画出学习曲线，观察训练误差和验证误差

超参数Hyperparameters

  1. Learning Rate是最重要的超参数，没有之一。
    网格搜索同样适用于深层神经网络的超参调整，每次选用少量的epoch测试。
  2. Learning Rate Decay是二阶超参，不需要一开始就引入。
    注意Learning Rate Decay和Adam的原理相悖，通常不兼容。

模型集成Ensemble

训练10个参数不同的模型，对预测结果投票，期望提升2%

特征重要性评估(Feature Importance Evaluation)

发表于 2018-05-19
字数统计: 446

特征对目标变量预测的相对重要性，可以通过决策树中使用特征作为决策节点的相对顺序来评估。
决策树顶部使用的特征，将对更多样本的最终预测决策做出贡献。
因此，可以通过每个特征对最终预测做出贡献的样本比例，来评估该特征的重要性。

通过对多个随机树中的预期贡献率取平均，可以减少这种估计的方差。

sklearn.ensemble模块包含两个基于随机决策树的平均算法：Random Forest 和 Extra-Tress.
这两种算法都是在构造过程中引入随机性来创建一组不同的分类器。

缺失值处理

发表于 2018-05-19
字数统计: 573

缺失值的处理方法分为三类：

  • 删除：删除含有缺失字段的样本
  • 修补：有多种方法
  • 不处理：直接使用不完整数据作为输入

修补方法

  1. 人工修补(filling manually)
    由了解数据的用户填写，偏离最小，填充效果最好。
    成本巨大，当数据规模大时不适用。

  2. 特殊值(Special values)
    将缺失值当做一种特殊的属性值来处理，不同于其他的任何属性值。例如’unknown’.
    可能导致严重的数据偏离。一般不推荐使用。

  3. 均值填充(Mean/Mode Completer)
    用现存数据的多数信息来推测缺失值。
    数值型字段，使用其他样本在该字段上的均值；
    非数值型字段，使用出现次数最多的值。

  4. 热卡填充(Hot deck imputation)
    在完整数据中找到一个最相似的对象，然后用这个最相似对象的值填充。
    缺点在于难以定义相似标准。

  5. 聚类填充(clustering imputation)
    首先确定与缺失样本距离最近的k个样本，然后用k个样本的均值填充。

  6. 所有可能的值填充(all possible values)
    使用空缺属性值的所有可能的取值来填充。
    计算代价大，可能的测试方案多。

  7. 组合完整化方法(Combinatorial Completer)
    使用所有可能的属性取值来试，从中选择最好的一个作为填补的属性值。

  8. 回归(Regression)
    基于完整的数据集，建立回归方程。将缺失记录的已知属性值代入方程，得到缺失值的回归估计值。
    当变量不是线性相关时，会导致有偏差的估计。

  9. 极大似然估计(Max Likelihood)
    通过观察数据的边际分布可以对未知参数进行极大似然估计。
    最常采用的计算方法是期望值最大化(Expectation Maximization, EM)。
    适用大样本，计算很复杂。

  10. 多重插补(Multiple Imputation)
    来源于贝叶斯估计，认为待修补的值是随机的，来自于已观测到的值。
    具体实践上通常是估计出待插补的值，然后加上不同的噪声，形成多组可选插补值。

Imbalanced Classes

发表于 2018-05-19
字数统计: 347

不平衡的数据集是指不同类别的样本数差距过大。
例如二分类问题中90个负样本，10个正样本。使用Accuracy会得到虚假的高准确率。

应对方案

1. 收集更多数据

数据分布可能改变，另外收集更多的小类别样本对后续重采样也有帮助。

2. 更换Metric指标

推荐使用：

  • Confusion Matrix
  • Precision
  • Recall
  • F1 Score

或者

  • Kappa
  • ROC Curve
3. 重采样

对现有数据集重采样的方法有二：

  • over-sampling: 向小类别样本集中添加样本的拷贝
  • under-sampling: 从大类别样本集中删除样本
4. 合成样本

生成实际上并不存在的新样本，最常用的方法是SMOTE：Synthetic Minority Over-sampling Technique

5. 使用不同的学习器

Tree-Based Model通常也能在不平衡数据集上表现良好。

6. 增加惩罚项

提高小类别样本分类错误时的cost，使分类器更注重这些样本。

7. 另一种数据洞察

将小类别样本分类问题当做另一种学习任务来处理：

  • 异常检测(Anomaly Detection): 将小类别样本当做异常点，使用异常检测的算法来识别
  • 变化检测(Change Detection): 类似于寻找用户行为的变化
8. 创造性

将问题分解为较小的子问题集

Keras回调函数Callbacks

发表于 2018-05-16
字数统计: 638

回调函数是一组在训练的特定阶段被调用的函数，可以用来观察网络内部的状态。
回调函数通常被传递到模型的 fit() 方法中。

Callback

keras.callbacks.Callback()
回调函数的抽象类，定义新的回调函数必须继承此类

BaseLogger

keras.callbacks.BaseLogger()
Callback that accumulates epoch averages of metrics.
计算metrics指标的累加epoch平均值。
在每个keras模型中自动调用。

History

keras.callbacks.History()
History对象即为fit() 方法的返回值。
在每个keras模型中自动调用。
History.history是一个字典，键值包括[‘acc’,’loss’,’val_acc’,’val_loss’,’lr’]等。
value是长度=epoch的列表，即每个epoch记录一次数据。

TensorBoard

当使用TensorFlow作为后台时，也可以在CallBack函数里使用TensorBoard.

启动之后，通过浏览器打开http://localhost:6006查看

ReduceLROnPlateau

keras.callbacks.ReduceLROnPlateau(monitor=’val_loss’, factor=0.1, patience=10, verbose=0, mode=’auto’, epsilon=0.0001, cooldown=0, min_lr=0)
当评价指标不再提升时，减少学习速率。
参数

  • mointor: 要监测的评价指标
  • factor: 减少因子，更新方式lr = lr × factor
  • patience: 经过多少个epoch而评价指标不变时，减少学习率的动作会被触发
  • mode: ‘auto’,’min’,’max’, 评价指标的正常变动方向
  • epsilon: 小于该阈值，则认为评价指标“无变化”
  • cooldown: 学习率减少后，经过多少个epoch后才会再次触发
  • min_lr: 学习速率下限

EarlyStopping

keras.callbacks.EarlyStopping(monitor=’val_loss’, min_delta=0, patience=0, verbose=0, mode=’auto’)
当评价指标不再提升时，中止训练。
参数

  • monitor: 要监测的评价指标
  • min_delta: 提升的最小阈值
  • patience: 经过多少个epoch而评价指标不变时，触发动作
  • verbose: 信息展示模式
  • mode: ‘auto’,’min’,’max’

ModelCheckpoint

keras.callbacks.ModelCheckpoint(filepath, monitor=’val_loss’, verbose=0, save_best_only=False, save_weights_only=False, mode=’auto’, period=1)
在每个epoch后将模型保存到文件。
参数

  • filepath: 文件路径，可以包含格式，例如weights.{epoch:02d}-{val_loss:.2f}.hdf5
  • save_best_only: 是否只保存在验证集上性能最好的模型
  • save_weights_only: ‘True’只保存权重，’False’保存整个模型(包括结构、配置等)
  • period: 两次保存之间间隔的epoch数

Xception

序贯模型是多个网络层的线性堆叠。

快速开始

构造
可以通过传递一个layer的list来构造模型，也可以通过add()方法将Layer逐个加入。

输入
Sequential的第一层需要接受一个输入数据shape的参数，后面的各个层可以自动推导出中间数据的shape.
为第一层指定输入数据shape的方法包括：

  • input_shape: tuple类型，不包括batch大小
  • input_dim: 某些层支持该参数
  • batch_size: 某些层需要指定固定大小的batch_size

编译
训练模型之前，需要通过compile对模型编译。
compile(self, optimizer, loss, metrics=None, sample_weight_mode=None)

  • optimizer: 优化器，必须
  • loss: 目标函数，必须
  • metrics: 评价指标，可选
  • sample_weight_mode: 默认”None”按样本赋予权值；”temporal”按时间步赋予权值

训练
fit(self, x, y, batch_size=32, epochs=10, verbose=1, callbacks=None,
validation_split=0.0, validation_data=None, shuffle=True,
class_weight=None, sample_weight=None, initial_epoch=0)

  • x: 单个输入numpy array，多个输入是list of numpy array
  • y: numpy array
  • batch_size: 优化一步的样本数
  • epochs: 训练的总轮数
  • verbose: 日志显示，0不输出，1输出进度条，2每个epoch输出一行记录
  • callbacks: list，其中的元素是Callback函数
  • validation_split: float(0~1)，验证集的比例
  • validation_data: 指定验证集(X,y)，将覆盖validation_split
  • shuffle: 是否随机打乱顺序
  • class_weight: 字典，不同类别映射为不同的权重
  • sample_weight: 样本权重，numpy array
  • initial_epoch: 从指定的epoch开始训练，用于继续之前的训练过程

其他方法

  • pop(self): 弹出模型最后的一层，无返回值

噪声层用于给数据附加随机的噪声，用以克服过拟合。
也是一种正则化，仅在训练时使用。

GaussianNoise

添加均值为0，标准差为stddev的高斯噪声。

乘以将均值1，标准差为$\sqrt{\frac{rate}{1-rate}}$ 的噪声。

使输入数据的均值和方差保持不变，保持数据的自规范性。

噪声的标准差也是$\sqrt{\frac{rate}{1-rate}}$

Keras规范化层BatchNormalization

发表于 2018-05-04
字数统计: 240

在每次迭代的batch里，对上一层的输出规范化。算法分两步：

  1. Normalize: 将输出变为均值0，标准差1；
  2. Scale and Shift: 加入新的均值和方差，避免失去学习到的信息

用公式表示算法流程：
$$\begin{align}
\mu_B &\leftarrow \frac{1}{m} \sum_{i=1}^m x_i &&\text{batch mean} \\
\sigma_B^2 &\leftarrow \frac{1}{m} \sum_{i=1}^m \;(x_i-\mu_B)^2 &&\text{batch variance} \\
\hat{x_i} &\leftarrow \frac{x_i - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}} &&\text{normalize} \\
y_i &\leftarrow \gamma\hat{x_i} + \beta \equiv BN_{\gamma,\beta}\;(x_i) &&\text{scale and shift}
\end{align} $$

代码

  • axis: 指定要规范化的轴
  • momentum: 移动平均和移动方差的冲量，公式如下：
    moving_mean = momentum × moving_mean + (1 - momentum) × batch_mean
    moving_var = momentum × moving_var + (1 - momentum) × batch_var
  • epsilon: 加到方差里的小浮点数，避免除数为0
  • center: 若True，则给规范化后的tensor加上偏移量beta
  • scale: 若True，则乘以gamma
