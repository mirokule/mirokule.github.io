Unity-Input

发表于 2016-08-23

Input类是处理输入的接口。

Key与物理按键对应，如键盘、鼠标、摇杆，通过名称或编码KeyCode访问。
Button与虚拟按键对应，如“Fire 1”，通过名称访问。

鼠标

  • mousePosition: 鼠标位置，Vector3(x, y, 0)

  • GetMouseButtonDown(n): 鼠标按下的第一帧返回true，0左键，1右键，2中键

  • GetMouseButtonUp(n): 鼠标松开的第一帧返回true

  • GetMouseButton(n): 鼠标按下期间持续返回true

  • GetAxis(“Mouse X”): 得到一帧内鼠标在水平方向的移动距离

  • GetAxis(“Mouse Y”): 得到一帧内鼠标在垂直方向的移动距离

键盘

  • GetKey
  • GetKeyDown
  • GetKeyUp
  • GetAxis(“Horizontal”)
  • GetAxis(“Vertical”)

触摸

  • GetTouch
  • Touches 当前所有触摸状态列表
  • touchCount 触摸的点数

返回的是数据保存在Touch结构体中，Touch结构体的变量如下：

  • fingerid 唯一索引
  • position
  • deltaPosition
  • deltaTime
  • tapCount
  • phase 状态描述

Phase的状态如表

  • Began 手指刚触碰屏幕
  • Moved 在屏幕上移动
  • Stationary 自上一帧没有移动
  • Ended 离开屏幕
  • Canceled 系统取消了跟踪触摸

Unity-Transform

发表于 2016-08-23
字数统计: 312

每个GameObject都有一个Transform，记录了对象的位置、大小、旋转等信息。

成员变量：

  • position: 世界坐标系中的位置
  • eulerAngles: 世界坐标系中，以欧拉角表示的旋转

  • rotation: 世界坐标中，以四元数表示的旋转

  • localPosition: 父对象局部坐标系中的位置

  • localEulerAngles: 父对象局部坐标系中的欧拉角

  • localRotation: 父对象局部坐标系中四元数表示的旋转

  • up: 对象在世界坐标系中的上方向, y轴？

  • right: 对象在世界坐标系中的右方向, x轴？

  • forward: 对象在世界坐标系中的前方向, z轴？

  • parent: 父对象的transform组件

  • childCount: 子对象的数量

成员函数：

  • Translate: 按指定的方向和距离平移
  • Rotate: 按指定的欧拉角旋转
  • RotateAround: 按指定的轴和角度旋转

  • LookAt: 旋转使得自身的前方向(forward)指向目标位置

  • TransformDirection: 将一个方向从局部坐标系变换到世界坐标系

  • InverseTransformDirection: 反向
  • TransformPoint: 将一个位置从局部坐标系变换到世界坐标系

  • InverseTransformPoint: 反向

  • DetachChildren: 与所有子物体解除父子关系

  • Find: 按名称查找子对象
  • IsChildOf: 判断是否为指定对象的子对象

Unity-Quaternion

发表于 2016-08-18
字数统计: 271

Rotation

旋转本质上需要知道 1个向量 + 1个角度，共4个值。有3种表示方法：

  • Matrix(矩阵): 使用 4*4 矩阵，共16个值，计算时有额外的时间和空间消耗；

  • Euler(欧拉): 使用3个值(xRotation, yRotation, zRotation)，是一系列坐标轴旋转的组合，按顺序沿各个轴旋转；会带来万向节锁问题。
    Gimbal Lock(万向节锁)：当欧拉旋转的第二次旋转角度为90°时，第三个轴就被转到了第一个轴相同的方向，因此丢失了一个自由度。

  • Quaternion(四元数): 本质上是一个高阶复数，即 x = a + bi + cj + dk，效率更高

    rigidBody.rotation = Quaternion.Euler(0.0f, 0.0f, rigidBody.velocity.x * -1);

乘法

  1. quaternion quaternio，例如q = q1 q2，表示将一个点先q1旋转，再q2旋转
  2. quaternion Vector3，例如p = q v，表示将向量v进行q旋转，得到旋转后的向量p.
    Quaternion必须位于乘号的左边。
//镜头到物体的距离
distance = Vector3.forward * 10.0f;
transform.position = target.position - distance
//有旋转
transform.position = target.position - rotation * distance

GKRandom

Randomization Distributions

  • Random: GKRandomDistribution(:)
  • Gaussian: GKGausianDistribution(:), good for making corridors
  • Shuffled: GKShuffledDistribution(:), does not repeat a number until all the numbers have been returned, good for making rooms

Randomization Sources

  • Linear Congruential: GKLinearCongruentialRandomSource(), least computation and randomness
  • ARC4 random: GKARC4RandomSource(), the default source
  • Mersenne Twister: GKMersenneTwisterRandomSource(), most computation and randomness

GKGraph is a collection of locations in game scene, which are used in path finding. Subclass of GKGraph includes:

  • GKGridGraph: A list of valid locations on a two-dimensional grid. Entities are only allowed to travel between these grid locations. Example as Pac-Man.
