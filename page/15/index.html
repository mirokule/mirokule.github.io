Instantiate View Controller

If the view controller is made by code, use 

let popoverViewController = PopOverViewController(...)

If the view controller is made in storyboard, use 

let popoverViewController = storyboard?.instantiateViewControllerWithIdentifier("scoreView") as! PopUpViewController

UIModalPresentationStyle

  • .Popover present as a modal view controller on the compact-width screens, but as a popover on regular-width screens

  • .OverFullScreen displays the presented view over the full screen, let underlying views show through

  • .OverCurrentContext displays the presented view only over the parent view controller’s content, let underlying views show through

UIModalTransitionStyle

  • .CoverVertical
  • .FlipHorizontal
  • .CrossDissolve
  • .PartialCurl

Important

SKNode will continue running SKAction after being set position, so use

SKNode.removeAllActions()

to avoid unexpected moves.

Some Action

  • Move
  • Rotatae
  • Scale

  • reSize

  • setColor

  • setTexture

  • animateWithTexture

  • hide

  • fadein/out

  • removeFromParent

  • waitForDuration

  • runAction:onChildWithName

  • runBlock
  • customActionWithDuration

  • actionNamed

  • playSound

SKActionTimingMode

  • Linear: animation occurs evenly over its duration
  • EaseIn: begin slowly and then speed up
  • EaseOut: begin quickly and then slow as it completes
  • EaseInEaseOut: begins slowly, accelerates through middle, then slows again before completing

SpriteKit-Anchor Point and Position

发表于 2015-10-13
字数统计: 80

Position

A node’s position is its location in parent coordinate system.

Anchor Point

??A skscene‘s anchorPoint property determins the origin point in the scene.

A sprite node’s anchorPoint property determines which point in the frame is positioned at the sprite’s position.

Example

  1. Point = (80, 80), anchorPoint = (0.5, 0.5): the sprite puts its center on (80,80)

  2. Point = (80, 80), anchorPoint = (0, 1): the sprite puts its left-top point on (80,80)
