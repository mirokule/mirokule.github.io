1.如果没有标明成员函数或成员变量的访问权限级别，
C++中，struct中默认的是public，class中默认的是private
C#中，struct和class中都是private的。

2.struct定义的是值类型，值类型的实例在栈上分配内存，由编译器自动分配和释放
class定义的是引用类型，引用类型的实例在堆上分配内存，由用户分配和释放

3.C#可以定义Finalizer和Dispose方法来释放资源，类似于C++的析构函数，但运行时机不确定
静态构造函数，在类型第一次被使用前自动调用，且保证只调用一次

4.C++支持多继承，C#类只能继承一个基类中的实现但可以实现多个接口。

5.C# 中没有全局方法和全局变量：方法和变量必须包含在类型声明（如 class 或 struct）中。

6.C# 不能重写非虚方法或静态方法。能重写的基方法必须是 virtual、abstract的。

  • virtual：基类必须有实现，子类可以重写或不重写，用于“可以重写的方法”
  • abstract：基类必须没有实现，子类必须有实现，类似接口，用于“必须重写的方法”
  • override：子类重写的方法，前缀必须加上override

7.接口是抽象的集合，不能存在实现的功能
抽象类不能被实例化，存在抽象方法，具有类的其他特性；可以有已经实现的方法

8.private类型完全私有，不能被子类继承，只有当前类中的成员能访问；
protected类型受保护，能被子类继承，只有当前类的成员、子类成员能访问；
public类型完全公开，能被子类继承，大家都可以访问。

地图划分

1.Grid (方格)

将搜索区域划分成了方形网格。
没有障碍物的地方画上方格，选取方格中心作为可以行走的点。

优点：划分简单，容易结合寻路算法使用。
缺点：移动是不连续的；网格数较多时需要资源计算和存储。

2.Navigation Mesh(导航网格)

利用了凸多边形的性质：在凸多边形上的一个点走到另一个点，路径一定在多边形内部。

生成步骤：
（1）通过地图的边界点和障碍物的边界点，将地图划分成许多三角形；
（2）将相邻的三角形合并形成凸多边形；
（3）将凸多边形继续不断合并为凸多边形；
（4）将凸多边形每条边的中点放置路径点，将每个多边形的路径点连线。

优点：降低搜寻计算量，移动更加自然。
缺点：实现复杂。

Surface Effector 2D

在collider表面提供一个切向力，类似于传送带。

Platform Effector 2D

Buoyancy Effector 2D

模仿液体的浮力、流动等。

Area Effector 2D

在区域内定义作用力场，当目标进入时将受到作用力。

Stores and accesses player preferences between game sessions.

Static Functions

  • DeleteAll Removes all keys and values from the preferences. Use with caution.

  • DeleteKey Removes key and its corresponding value from the preferences.

  • HasKey Returns true if key exists in the preferences.

  • GetFloat Returns the value corresponding to key in the preference file if it exists.
  • GetInt Returns the value corresponding to key in the preference file if it exists.

  • GetString Returns the value corresponding to key in the preference file if it exists, return defaultValue if not.

  • Save Writes all modified preferences to disk.

  • SetFloat Sets the value of the preference identified by key.
  • SetInt Sets the value of the preference identified by key.
  • SetString Sets the value of the preference identified by key.

Unity-SceneManager

发表于 2016-10-05
字数统计: 173

Scene management at run-time.

Variables

  • sceneCount The total number of currently loaded scenes.
  • sceneCountInBuildSettings Number of scenes in Build Settings.

Functions

Get
  • GetActiveScene Gets the currently active scene.
  • GetSceneAt Get the scene at index in the SceneManager’s list of added scenes.
  • GetSceneByBuildIndex Get a scene struct from a build index.
  • GetSceneByName Searches through the scenes added to the SceneManager for a scene with the given name.
  • GetSceneByPath Searches all scenes added to the SceneManager for a scene that has the given asset path.
    Load
  • LoadScene Loads the scene by its name or index in Build Settings.
  • LoadSceneAsync Loads the scene asynchronously in the background.
  • UnloadSceneAsync Destroyes all GameObjects associated with the given scene and removes the scene from the SceneManager.
    Create
  • CreateScene Create an empty new scene at runtime with the given name.
  • SetActiveScene Set the scene to be active.
  • MergeScenes This will merge the source scene into the destinationScene.
  • MoveGameObjectToScene Move a GameObject from its current scene to a new Scene.
    Example
    // restart current scene
    SceneManager.LoadScene (SceneManager.GetActiveScene ().name);
