Imbalanced Classes

发表于 2018-05-19
字数统计: 347

不平衡的数据集是指不同类别的样本数差距过大。
例如二分类问题中90个负样本，10个正样本。使用Accuracy会得到虚假的高准确率。

应对方案

1. 收集更多数据

数据分布可能改变，另外收集更多的小类别样本对后续重采样也有帮助。

2. 更换Metric指标

推荐使用：

  • Confusion Matrix
  • Precision
  • Recall
  • F1 Score

或者

  • Kappa
  • ROC Curve
3. 重采样

对现有数据集重采样的方法有二：

  • over-sampling: 向小类别样本集中添加样本的拷贝
  • under-sampling: 从大类别样本集中删除样本
4. 合成样本

生成实际上并不存在的新样本，最常用的方法是SMOTE：Synthetic Minority Over-sampling Technique

5. 使用不同的学习器

Tree-Based Model通常也能在不平衡数据集上表现良好。

6. 增加惩罚项

提高小类别样本分类错误时的cost，使分类器更注重这些样本。

7. 另一种数据洞察

将小类别样本分类问题当做另一种学习任务来处理：

  • 异常检测(Anomaly Detection): 将小类别样本当做异常点，使用异常检测的算法来识别
  • 变化检测(Change Detection): 类似于寻找用户行为的变化
8. 创造性

将问题分解为较小的子问题集

VC维

发表于 2018-04-28
字数统计: 766

假设空间H作用于数据量为N的样本集D，h的数量看上去是无穷的，但真正有效(effective)的h的数量确实有限的，这个数量记做$m_H(N)$.
H中每一个h作用于D都能算出一个$E_{in}$来，共有$m_H(N)$个不同的$E_{in}$

Dichotomy

二分类问题中给定样本集D，如果D是线性可分的，就把一种分类方式称作一个Dichotomy(对分)。
一个Dichotomy中包含了所有可以分出这个局面的h，相当于把假设空间H里的所有h按照不同的分类结果来分组。
样本集D上的所有划分组成了dichotomy set.

Growth Function

对特定样本数N，可以有多个采样集D，每个D对应不同的dichotomy set。
将这些dichotomy set的最大值，定义为N的成长函数Growth Function: m(N)。
增长函数表示假设空间H对D中样本赋予标记的最大可能结果数。

Shatter

Shatter(打散)指假设空间H能够实现D中全部示例的对分，即增长函数=$2^N$
另一种定义：对于一个假设空间H，如果存在m个数据样本能够被假设空间H中的函数按所有可能的$2^h$种形式分开，则称假设空间H能够把m个数据样本打散。

Break Point

使Growth Function < $2^H$成立的N的最小值。
若k是Break Point，那么k+1, k+2, …对应的Growth Function都会小于$2^N$

VC Bound

将成长函数的上界，记为B(N,k)，意为：maximum possible m_H(N) when break point = k.

  • B(2,2) = 3
  • B(3,2) = 4
  • B(N,1) = 1
  • B(N,k) = $2^N$ for N<k; $2^N-1$ for N=k

最后利用数学归纳法，可以证明得到下面的bounding function(N>k)
$$ m_H(N) \leq \sum_{i=0}^{k-1} \binom{N}{i} $$


如果假设空间存在有限的break point，那么$m_H(2N)$会被最高幂次为k-1的多项式上界给约束住。
随着N的逐渐增大，指数式的下降会比多项式的增长更快，所以此时VCBound是有限的。
更深的意义在于，N足够大时，对H中的任意一个假设h，$E_{in}(h)$都将接近于$E_{out}(h)$，这表示机器学习可行的第一个条件是有可能成立的。

Vapink-Chervonenkis Dimension

假设空间H的VC维是能被H打散的最大的样本集的大小，记做$d_{vc}(H)$。
若对任意数目的数据样本都有函数能将它们shatter，则假设空间的VC维为无穷大。
VC维的大小只与模型和假设空间有关，反应了假设空间H的强大程度(powerfulness)。VC维越大，H越强，能打散更多的点。

根据定义，可以得到一个明显的结论：
$$ k = d_{vc}(H) + 1$$

一个实践规律：VC维与假设参数w的自由变量数目大约相等。
模型较复杂($d_vc$较大)时，需要更多的训练数据。实际经验里，一般$N\approx 10\times d_{vc}$

这个函数的本意是获取已存在的变量，因而必须指定变量名称。当变量不存在时，将会创建一个。

变量集合

默认情况下，每个 tf.Variable 都放置在以下两个集合中：

  • tf.GraphKeys.GLOBAL_VARIABLES：可在多个设备间共享
  • tf.GraphKeys.TRAINABLE_VARIABLES：将计算梯度并迭代更新

自定义的集合不需要显式创建，直接用在add_to_collection()方法中即可。

变量初始化

变量在使用前必须初始化。
高级框架如 tf.estimator.Estimator, tf.contrib.slim, Keras 等在训练前会自动初始化变量；而使用低级别API编程时需要手动初始化变量。

要在训练前一次性初始化所有可训练变量，使用 tf.global_variable_initializer()。该函数会返回一个op，负责初始化 tf.GraphKeys.GLOBAL_VARIABLES 集合中的所有变量。运行这个op就会将集合里的所有变量初始化。

注意：tf.global_variable_initializer 并不会指定变量的初始化顺序。如果某个变量的初始值取决于另一变量，那么初始化时很有可能出现错误。

要为变量赋值，可以使用 assign, assgin_add 以及 tf.Variable 类中的友元。

霍夫丁不等式与机器学习

发表于 2018-04-28
字数统计: 820

霍夫丁不等式

概率论中的霍夫丁不等式(Hoeffding’s Inequality)用来衡量抽样与整体的分布偏差：
$$ P[\left| v-\mu \right| > \epsilon] \leq 2 e^{-2N\epsilon^2}$$
若抽样样本数N足够大，抽样分布v就很可能接近整体分布$\mu$

固定假设h(x)

引入到机器学习领域，固定假设$h(x)$，将待抽样的小球定义为：

  • 橙色：若$h(x) \neq f(x)$
  • 绿色：若$h(x) = f(x)$

那么每个训练集$D$可以看做是一次抽样，$E_{in}$是训练集的误差，$E_{out}$是整体误差。
根据霍夫丁不等式，如果抽样样本数N足够大，那么$E_{in}$背离$E_{out}$的几率就很小。
$$ P[\left| E_{in}(h) - E_{out}(h) \right| > \epsilon] \leq 2 e^{-2N\epsilon^2}$$

真实的机器学习

在真实的机器学习问题中，$h(x)$不是固定的，而是由算法A选择的。
虽然单个$h(x)$的$E_{in}$和$E_{out}$不容易背离，但是算法A要检验多个$h(x)$，那么算法A就可能会遇到$E_{in}$和$E_{out}$背离的情况。
这种使$h(x)$的$E_{in}$和$E_{out}$背离的抽样集D称作 BAD Sample。


表中$D_{1126}$ 这样的抽样集作为训练数据是比较优质的。
如果抽样集D对某个$h(x)$来说是Bad Sample，那么D对整个假设空间H来说也是Bad Sample。因为算法A使用抽样集D作为训练数据，来对假设空间H筛选时，不能找到最优的$h(x)$.
给定抽样集D，它是假设空间H的Bad Sample的概率为：
$$ \begin{split}
P_D[Bad\; D]&= P_D[Bad\; D for\; h_1\; or\; h_2\; or\;…\;or\; h_M] \\
&\leq P_D[Bad\; D\; for\; h_1] + P_D[Bad\; D\; for\; h_2] +…+ P_D[Bad\; D\; for\; h_M] \\
&\leq 2e^{-2N\epsilon^2} + 2e^{-2N\epsilon^2} +…+ 2e^{-2N\epsilon^2} \\
&\leq 2Me^{-2N\epsilon^2}
\end{split} $$

也就是说，假设空间H中备选假设的数量M越少，样本数据量N越大，那么抽样集D成为坏样本的概率就越小。
在一个可接受的概率水平上，算法A只需要挑出$E_{in}$最小的那个$h(x)$作为g就可以了。
注意：union bound提升了概率上限，事实上有很多重叠，并不是独立事件。

M的取值

假设空间H里的备选假设数量M，会在两方面影响机器学习：

  • 能否确保训练样本和测试样本的错误率足够接近：由上文公式推断
  • 能否使训练样本上的错误率足够小：g要接近f
M太小 M太大
$E_{in}$接近$E_{out}$ 能，BAD概率较小 不能，BAD概率较大
$E_{in}足够小$ 不能，选择太少 能，选择足够多

解决思路：找到一个更合理的$m_H$替代M。因为union bound提升了概率上限，使我们过高估计了BAD概率。
对样本数为N的训练集，我们可以将M个备选假设分为若干类，每一类里的假设$h(x)$对N个样本的分类结果都相同。
类别的总数记做effective(N)。可以证明$effective(N)\leq 2^N$。
这样就可以用effective(N)替代M，概率不等式的右侧不再是一个无上限的参数。

TensorBoard是TensorFlow内置的可视化工具，可以在训练过程中以可视化的方法监控模型内部发生的一切。
TensorBoard的功能包括：

  • 在训练过程中以可视化的方式监控指标
  • 将模型结构可视化
  • 将激活和梯度的直方图可视化
  • 以三维的形式研究嵌入

TensorBoard通过读取TensorFlow的事件文件来运行。
TensorFlow的事件文件包含运行TensorFlow时生成的总结数据。

步骤

1. 标注

首先在计算图中想要输出的节点里标注总结指令(summary operations)。
例如记录学习速率如何变化，就向节点附加 tf.summary.scalar;
想要显示某层的权重分布，就附加 tf.summary.histgram 来收集数据。

添加的summary operations并没有被计算流程依赖，因而不会自动运行，需要手动逐个运行。
因而使用 tf.summary.merge_all 来将这些操作合并为一个op，然后执行该合并的总结op，得到所有总结数据。
总结数据是一个序列化的 Summary protobuf 对象。

示例

总结指令包括：

  • tf.summary.tensor_summary
  • tf.summary.scalar
  • tf.summary.histogram
  • tf.summary.audio
  • tf.summary.image
  • tf.summary.merge
  • tf.summary.merge_all
2. 保存

将总结数据保存到硬盘，需要将protobuf对象传递给 tf.summary.FileWriter.
这将在当前目录中生成一个event文件：events.out.tfevents.{timestamp}.{hostname}

  • logdir: 所有事件都会写到这个指定的目录下
  • graph: 如果非空，则将图与张量形状信息一起可视化;等价于add_graph()

其中，logdir指向FileWriter将数据序列化的目录。
如果logdir目录下有子目录，子目录包含序列化数据，那么TensorBoard会将这些数据都可视化。

4. 查看

TensorBoard运行后，就可以在浏览器中打开 localhost:6006 以查看。

TensorFlow会话Session

发表于 2018-04-27
字数统计: 527

TensorFlow使用 tf.Session 类来表示客户端程序(通常为Python)与C++运行时之间的接口。
tf.Session 对象通常称为会话，会封装TensorFlow运行时的状态，并运行TensorFlow指令。
如果说 tf.Graph 像一个.py文件，那么 tf.Session 就像一个可执行的python.

创建tf.Session

tf.Session.init 可接受三种可选的参数：

  • target: 指定grpc://网址以使用TensorFlow服务器，留空则仅使用本地计算机的设备
  • graph: 默认使用当前的默认计算图，也可以指定其他的 tf.Graph
  • config: 指定一个 tf.ConfigProto 以便控制会话的行为。部分配置选项包括：
    • allow_soft_placement：为True则启用“软”设备分配算法
    • cluster_def：使用分布式TensorFlow时，指定要在计算中使用的机器、映射
    • graph_optionis.optimizer_options：对计算图执行优化的控制
    • gpu_options.allow_growth：为True则使GPU内存分配工具逐渐增加分配的内存量，而不是在启动时分配掉大多数内存

执行指令

tf.Session.run 方法是一种运行 tf.Operation 或对 tf.Tensor 求值的主要机制。

tf.Session.run也可以接受Feed字典，该字典是从 tf.Tensor 对象(通常是 tf.placeholder 张量)到张量的值(Python标量、列表或者Numpy数组)的映射。张量的值会在运行时被替换。

tf.Session.run 也接受可选的参数：

  • options参数：指定与调用有关的选项
  • run_metadata参数：收集与执行有关的元数据

TensorFlow计算图Graph

发表于 2018-04-27
字数统计: 894

计算图是排列成一个图的一系列TensorFlow指令。由两种类型的对象组成：

  • 指令(op)：图的节点。说明消耗和生成张量的计算。
  • 张量：图的边，流经图的值。


图中的每个指令都拥有唯一的名称。张量是根据生成它们的指令命名的，后面跟着输出索引，例如”add:0”

动态计算图

在tf1.0时代，采用的是静态计算图，需要先创建计算图，然后开启会话seesion，显式执行计算图；而在tf2.0时代，采用的是动态计算图，每使用一个算子，该算子会被动态加入到隐含的计算图中，立刻执行得到结果，无需开启session。

动态计算图的好处是方便调试，像python原生代码一样。缺点是执行效率低，会有多次Python与tf底层c++进程之间的通信；同时也会失去静态计算图对计算步骤的优化。

解决方法是用装饰器@tf.function将普通Python函数转换成对应的tf计算图。运行该函数就相当于在tf1.0中用session执行代码。使用@tf.function构建静态图的方式叫做Autograph.

tf.Graph

tf.Graph 包含两类相关信息：

  • 图结构：图的节点和边，指明了各个指令组合在一起的方式，但不规定它们的使用方式。
  • 图集合：TensorFlow提供了一种通用机制，将创建的对象加入特定的集合中，并与键关联。

构建

TensorFlow提供了一个默认图，此图是上下文中所有API函数的明确参数。
API函数可以构建新的 tf.Operation (节点)和 tf.Tensor (边) 对象并将它们添加到 tf.Graph 实例。例如：

  • tf.constant(42): 创建单个 tf.Operation，生成值42，并返回一个表示常量值得 tf.Tensor
  • tf.matmul(x,y): 创建单个 tf.Operation，将 tf.Tensor 对象x和y的值相乘，并返回表示结果的 tf.Tensor
  • tf.train.Optimizer.minimize: 将指令和张量添加到计算梯度的默认图，返回一个 tf.Operation

多个图
使用 tf.Graph.as_default 上下文管理器，安装另一个 tf.Graph 来作为默认图

命名

tf.Graph 对象为其包含的 tf.Operation 对象定义一个命名空间。
TensorFlow会自动为图中的每个指令选择唯一名称，可以由用户指定。
tf.Tensor 对象以输出张量的 tf.Operation 命名，名称格式为”<OP_NAME>:<i>”，其中：

  • OP_NAME 是生成该张量的指令的名称
  • i 是一个整数，表示该张量在指令的输出中的索引

改写指令名称的方法有二：

  • name：每个创建operation或返回tensor的API函数都可以接受name参数
  • tf.name_scope：向特定上下文中创建的所有指令添加名称范围前缀。名称范围前缀用”/“分隔，重复的将使用”_1”、”_2”区分

Sequence to Sequence

发表于 2018-04-25
字数统计: 469

经典的N-to-N RNN要求序列等长，然而现实中的大部分问题序列都不是等长的，比如机器翻译.

Sequence-to-Sequence


如图，使用一个RNN网络将输入X转为编码c向量(可以是隐藏层参数、输出层变换等)，再用另一个RNN网络使用c向量作为输入，得到最终输出Y。
整个网络可以看做是求条件概率$ P(y^{<i>},…,y^{< T_y >}|x) $.

问：可以使用贪心法吗？
答：在每个时刻t，用贪心法求出的局部最优条件概率，不一定是全局最优条件概率。
例如机器翻译中，前n个词的最优翻译，不一定是整句话的最优翻译。

Beam Search

在解码的每个时刻，保留Top k 个最优解。当k=1时，算法退化为贪心法。
代价是要同时持有网络的k份拷贝。k越大，结果越好，运行越慢。

Length Normalization

目标函数中，小于1的概率p连续相乘会使得值过小：
$$ \prod_{t=1}^{T_y} P(y^{<t>}|x,y^{<1>},…,y^{<t-1>}) $$
所以取Log将乘法转为加法，目标函数变为
$$ Log\sum_{t=1}^{T_y} P(y^{<t>}|x,y^{<1>},…,y^{<t-1>}) $$
这两个目标函数都会倾向于选择较短的翻译，所以加入规范化项：
$$ \frac{1}{T_y^\alpha}Log\sum_{t=1}^{T_y} P(y^{<t>}|x,y^{<1>},…,y^{<t-1>}) $$

BLEU Score

全称BiLingual Evaluation Understudy，用于评估机器翻译的质量。
主要思想：机器翻译结果与若干条人工翻译的结果对比，越接近则效果越好。

Attention Model

当输入序列较长时，固定长度的编码c向量中难以包含所有序列的信息，使性能变差。
Attention机制的实现是在解码的每一步时，只考虑编码的若干个中间结果。

Word Embedding

发表于 2018-04-21
字数统计: 1,543

Word Embedding是要从数据中自动学习到输入空间到Distributed Representation空间的映射f.

意义

单词的One-hot编码失去了单词词意之间的联系。
例如man和woman的距离应该小于man和apple的距离，而在one-hot编码中，每个单词之间的距离都一样。
为了解决这个问题，引入Word Embedding编码，可以看做是一种类别编码。
例如embedding矩阵有300个维度，每个单词在这300个维度上都有一个分数，同类单词的分数应该相近。

用E来表示embedding矩阵，那么有$E\cdot O_j = e_j$。
其中$O_j$是单词的one-hot编码，$e_j$是单词的embedding编码。
embedding编码不仅引入单词语意的相关性，还缩短了编码长度(onehot词汇表10k -> embedding维度300).
可以通过t-SNE等非线性变换方式将每个单词的300维embedding编码投射到2-D平面上，观察不同单词之间的距离。

如果要量化单词之间的相似性，方法有二：

  • cosine相似性：计算向量夹角$ \frac{u^Tv}{\mid\mid u-v \mid\mid} $
  • 差值距离：$\mid\mid u-v \mid\mid^2$
提升泛化能力

Word Embedding极大提升了训练数据较少时模型的泛化能力。
将模型的输入由单词的one-hot编码改为embedding编码后，即便模型遇到一个从未见过的新词，也可以通过embedding编码推断出新词的类别、近似语意。
注意：对输入数据较多的模型，比如翻译等，word embedding并不适用。

如何得到embedding矩阵

学习任务的输入是10k one-hot编码(x)，输出是300类别编码(y)。
在训练开始时我们并不知道Y是什么，没有样本，所以这是一个无监督学习。无监督学习的常用方法有二：

  1. 挖掘输入X之间的关系，比如聚类；
  2. 以目标输出Y构建一个新的任务：$g: y \rightarrow z$，其中z值已知。这样用数据X可以训练得到$k: x \rightarrow z$，也就是$z = g(f(x))$，中间的表达式$y=f(x)$是我们真正想要的目标。

Word Embedding的训练方法偏向2.

1. Word2Vec

Word2Vec模型中，主要有Skip-Gram和CBOW两种模型。
从直观上看，Skip-Gram是给定input word来预测上下文，而CBOW是给定上下文来预测input word。
本文以Skip-Gram为例。

Fake Task
如上节所述，对非监督学习，我们要训练一个目标是任务z的神经网络，而神经网络的隐藏层参数才是真正的目标y。
神经网络的设计要求(x,z)可知可训练，那么就取(输入单词，上下文单词)来作为训练数据。神经网络的Fake Task z是通过输入一个单词，预测哪些单词最可能在其上下文出现。输出是多个单词的出现概率，使用softmax。
直观理解：语意相近的单词，比如kitty和cat，在训练数据中和它们一起出现的上下文单词也相近，对应网络中softmax的输出相近，因而隐藏层参数也相近。

输入数据

以上图距离为例，依次选取每个单词作为input word。设定窗口宽带为2，则从上下文就能得到形如(input, output)的训练数据。
得到单词对后，将其one-hot编码。假设词汇表有10k单词，那么模型的输入、输出都是10k维向量。

隐藏层

假设要用300个特征来表示一个单词，那么隐藏层的参数维度是(10k × 300)，没有使用激活函数。
当神经网络训练完毕后，这个隐藏层的(10k × 300)参数矩阵就是我们的目标y——embedding矩阵。
在实际计算中，由于输入是10k维度的稀疏矩阵，且one-hot编码只有1维是数字1，因而并不会消耗资源计算矩阵乘法，而是直接选取参数矩阵的某行。

网络训练
(10k × 300)网络共有300万个参数，使用梯度下降法训练是相当慢的。更糟糕的是需要大量的样本数据来避免过拟合。
可以采用一些优化手段：

  • 将单词组合作为一个单词：比如New York
  • 对高频次单词抽样来减少训练样本数：the、to这种单词无助于语义理解，应以一定概率(或者全部)删除
  • Negative Sampling：对每个样本(input, output)，从不是output的9999个单词中选取k个作为负样本一起加入训练，这样每次更新只针对参数矩阵的k+1行。例如k=5，那么每次更新300×6=1800个参数。

2. GloVe

没有使用神经网络，而是统计规律。

共现矩阵
定义 $X_{i,k}$ 是单词i的上下文中出现单词k的次数；
$X_i = \sum_k X_{i,k}$ 是单词i的上下文中出现单词k的总次数；
$P_{i,k} = P(k|i) = \frac{X_{i,k}}{X_i}$ 是单词k出现在单词i上下文的条件概率；
$ratio_{i,j,k} = \frac{P_{i,k}}{P_{j,k}}$ 是单词k出现在单词i、单词j上下文的概率之比。

$ratio_{i,j,k}$ 单词j,k相关 单词j,k不相关
单词i,k相关 趋近1 很大
单词i,k不相关 很小 趋近1

目标函数
假设我们得到了词向量$v_i, v_j, v_k$，通过某种函数计算的$ratio_{i,j,k}$ 与上表统计规律相符的话，就意味着词向量中蕴含了共现矩阵的信息。
经过一系列推导，得到目标函数如下：
$$ J = \sum_{i,j}^Nf(X_{i,j})(v_j^Tv_j + b_i + b_j - log(X_{i,j}))^2 $$

其中权重函数$f(x)$应满足如下条件：

  1. $f(0)=0$：避免log(0)运算
  2. $f(x)$是非减函数：出现次数越多，权重越大
  3. $f(x)$对较大的x不能取太大的值：the、to这类单词

综合这3点将$f(x)$定义为：
$$ f(x)=
\begin{cases}
(\frac{x}{x_{max}})^\alpha , \;\;if \;x< x_{max} \\
1, \;\;if \;x\geq x_{max}
\end{cases} $$

RNN

循环神经网络(Recurrent Neural Networks, RNN)是一种通过隐藏层节点周期性的连接，来捕捉序列化数据中动态信息的神经网络。

经典RNN在预测<t>时刻的信息时，使用了前面<t-k>时刻的信息，没有使用后面<t+k>时刻的信息。
与之对应的变种：

  • Bidirectional RNN：数据双向流动，后面的信息也参与前面信息的预测
  • Deep RNNS：在每个时刻t，引入更多的层级。由于输入维度很大，通常不超过3层。

基本算法

前向传播


后向传播

结构类型

输入输出 输出单值 输出序列
输入单值 图片分类 生成诗歌/语音/音乐
输入序列 根据文章判断类型/情感 自然语言/语音处理

长期依赖问题(Long-Term Dependency)

描述：<t>时刻的预测，需要前面<t-N>时刻的信息参照，其中N是很大的数。
比如语言模型中，主语后面接了一个很长的定语从句，然后出现了谓词，要根据主语判断谓词的单复数。
解决方法：网络中引入memory cell($c^{<t>}$)，将<t>时刻的特定信息保存下来。

1. LSTM

  • $c^{t}$: 当前时刻的记忆内容
  • $\tilde{c}^{<t>}$: 当前时刻的备胎(由输入得到)，需进一步计算决定是否更新记忆
  • $a^{t}$: 当前时刻的输出，进入下一时刻的Cell，或者变换后作为输出$y^{<t>}$
  • $\Gamma_f^{<t>}$: Forget gate，为0时则忘记上一时刻的记忆内容$c^{<t-1>}$
  • $\Gamma_u^{<t>}$: Update gate，为1时则使用当前时刻的候选$\tilde{c}^{<t>}$来更新记忆$c^{<t>}$
  • $\Gamma_o^{<t>}$: Output gate，为1时则使用当前时刻的记忆$c^{<t>}$生成输出$a^{<t>}$
2. GRU


GRU作为LSTM的一种变体，将Forget Gate和Update Gate合并到了一起。
有些结构还引入了Relative Gate等改动。
构造简单，效果出众，训练速度快，近年非常流行。

CNN图像风格迁移

算法
输入两张图片，一张提供内容(C)，另一张提供风格(S)；
输出一张合成图片(G)，是内容C和风格S的叠加。
G初始为任意值，通过不断迭代G，使目标函数不断缩小。

目标函数
$$J(G) = \alpha J_{content}(C,G) + \beta J_{style}(S,G)$$

$J_{content}(C,G)$：

  1. 选择训练好的卷积网络
  2. 选择网络中部的隐藏层l(太深或太浅都不行)
  3. 对比两张图片的$a^{[l]}$
    $$ J_{content}(C,G) = \frac{1}{2} \left| a^{[l]C} - a^{[l]G} \right|^2 $$

$J_{style}(S,G)$：如何衡量”style”?

  1. 定义style = 不同channel之间的相关性，例如{锐角，直线，圆点}经常同时出现，{钝角，波浪线}很少同时出现
  2. 计算单张图片的相关性矩阵$G^{[l]}$，矩阵尺寸$n_c^{[l]} \ast n_c^{[l]}$
    定义 $a_{i,j,k}^{[l]}$ = activation at (i,j,k);
    那么 $G_{kk’}^{[l]} = \sum_{i=1}^{n_H^{[l]}} \sum_{j=1}^{n_w^{[l]}} a_{ijk}^{[l]} a_{ijk’}^{[l]}$
  3. 损失函数定义为两张图片相关性矩阵G的差异：
    $$ J_{style}^{[l]}(S,G) = \frac{1}{(2n_H^{[l]}n_W^{[l]}n_c^{[l]})^2}\left| G^{[l]S} - G^{[l]G} \right|^2$$
    $$ J_{style}(S,G) = \sum_l \lambda^{[l]} J_{style}^{[l]}(S,G) $$

CNN人脸识别

发表于 2018-04-18
字数统计: 446

问题分类：

  • Verification(验证)：1 to 1，给定一张人脸图片和一个身份，判断是否相符
  • Recognition(识别)：1 to K，给定一张人脸图片，判断此人是否在数据库里；
    由于要进行K次verification，对算法的准确性要求更高。

简单想法

输出层采用softmax，有多少员工就对应多少分类。
通过训练好的网络，给定一张图片，通过输出的类别来判断是谁。
问题

  1. 训练样本不足。不能获取员工的大量照片。
  2. 输出分类固定。当员工人数变动时，需要重新训练网络。

Siamese Network

原理：相似性判别函数
训练一个函数来判断两张图片的相似性：
d(img1, img2) = degree of difference between images
如果d(img1, img2)小于阈值，则认为两张图片是同一个人。

实现
将深层网络的输出作为图片的编码，$f(x^{(i)})$
不同的图片需要经过同样的网络编码，使距离可比较。
训练网络参数，使得相同图片的编码相近，不同图片的编码相离。

目标函数1：Triplet Loss
同时输入3张图片：Anchor、Positive、Negative
想要$d(A,P) + \alpha \leq d(A,N)$，其中$\alpha$是安全边界(margin)。
所以目标函数为
$$L(A,P,N) = max(d(A,P) - d(A,N) + \alpha, 0)$$

目标函数2：Logistic Loss
输入2张图片，判断编码是否相等，相等为1，不相等为0.
$$\hat{y} = sigmoid(\sum_{k=1}^{128} \left | f(x^{(i)})_k - f(x^{(j)})_k \right | + b) $$

训练
选择d(A,P)和d(A,N)相近的图片，加快模型学习效率。
即通过较难样本的学习，从而在较简单样本的测试中获得较好的性能。

CNN目标检测

目标检测问题可以分为以下几种：

  • 单目标定位：输出$y = [P_c, b_x, b_y, b_h, b_w, c_1, c_2, c_3, … c_n]$
  • Landmark检测：输出$(L_x, L_y)$
  • 多目标检测：识别并定位图中的多个目标物体
    • 滑窗、卷积滑窗法
    • YOLO
    • R-CNN、Fast R-CNN、 Faster R-CNN

1. 滑窗法(Sliding Windows Dectection)

  1. 训练模型。使用切好的车辆、行人图片作为训练数据。
  2. 将测试图片切成$k_1\ast k_1$片，将每一片分别作为模型输入，得到输出;
  3. 使用更大尺度将图片切成$k_2 \ast k_2$片，再次作为输入；
  4. 反复运算之后取得分最高的作为最终输出。

缺点：计算开销太大，共需模型运算$k_1^2 + k_2^2 + … + k_n^2$次。

卷积实现

直接输入测试图片，共用中间计算部分，可以极大提高滑窗算法的效率。
最终输出的每一方格对应滑窗法中的每一块切片。
缺点：对位置的判断不够精确

2. YOLO

YOLO(You Only Look Once)
训练数据：认为物体在中心点所在的格子，中心点格子负责检测物体，其他格子假装没有。
每个格子对应的输出是$y = [P_c, b_x, b_y, b_h, b_w, c_1, c_2, c_3, … c_n]$

Anchor-Box
解决一格内多目标的问题。
允许一格内出现多个识别框，将输出数据y拉长到n倍。
举例：
原输出长度为8，$y = [P_c, b_x, b_y, b_h, b_w, c_1, c_2, c_3]$
现允许一格内出现5个物体，则输出拉长到40，或者(5,8)，是原来5个输出的堆叠。

Non-Max Suppressioin
解决同一物体被多个格子识别的问题。

  1. 通过计算识别方框的IOU比例来判断重叠(常用0.5)
  2. 在重叠的方框中取$P_c$最大的一个，忽视其余

3. R-CNN

  1. R-CNN(Regions with CNN)的基本思想是将图片分区，按不同区域检索物体。类似不均匀划分的的滑窗法。主要缺点仍然是运算较慢。
  2. Fast R-CNN在识别中应用了卷积实现；
  3. Faster R-CNN进一步在分区(segmentation)中应用了卷积实现。

R-CNN系列是一种两步(先分区再识别)的算法，和YOLO的一步算法有差异。
截止到本文写作时，R-CNN的计算速度仍然慢于YOLO。

CNN使用经验

发表于 2018-04-17
字数统计: 199

提升CNN网络表现除了更大更深的结构外，还有以下技巧：

  • 借助迁移学习：训练深层网络通常较慢，借助相关问题已经训练好的网络，修改最后几层来适应本问题。之前训练好的参数会加快本次训练收敛的速度
  • Data Augmentation：通过图片处理，获得更多的训练数据
    变形：镜像、切片、旋转、加噪声等
    变色：颜色迁移, PCA颜色变换等

另外有些方法会消耗更多算力，不一定在实际生产中采用，但可以提高Benchmark或用于竞赛：

  • Ensembing: 训练多个神经网络然后集成
  • Multi-Crop at test time: 将测试图片切片(例如10-crop)，对每个切片分别预测
