Dot 点积

u.v = |u||v|cos<u,v> = ux * vx + uy * vy，得到标量

在计算机图形学和游戏编程中，点积的重要用途之一是计算向量在给定方向上的分量(投影)。

Proj u = (u.v) * v / (|v|*|v|)

另外用于计算夹角
点积大于0则两个向量的夹角小于90度,锐角; 点积等于0，直角; 点积小于0,钝角

Vector3 a = new Vector3(1,1,1);
Vector3 b = enw Vector3(1,5,1);

float result = Vector3.Dot(a,b);
float angle = Vector3.Angle(a,b); //夹角，返回[0 - 180]

Cross 叉积

仅当向量包含3个或更多分量时，叉积才有意义。

u*v = |u|*|v|*sin<u,v> * n

其中n是uv平面的单位法向量。
实际应用中，使用矩形形式计算单位法向量n，很少直接使用叉积。

// c = a x b，得到向量，向量的模为 |c| = |a||b|sin<a,b>，且向量c垂直于a、b组成的平面，a x b = - b x a(左/右手定则?)
Vector3 c = Vector3.Cross(a,b);

// 判断顺时针、逆时针方向，指定XZ平面，考察叉积的Y轴分量
if (c.y > 0)
{ ... }

// 返回夹角，给angle标明方向

float signed_angle = angle * sign;

  • World Space: 世界坐标，在场景中添加物体的坐标系，transfrom.position

Unity-GI

发表于 2016-09-08
字数统计: 462

Unity中，实时照明(Realtime Lighting)的光线不会反射，只有投射光锥体范围内的对象表面才有光源影响，阴影是全黑的。
因此，引入Global Illumination全局照明，在直接光照的基础上，考虑了间接光照。

只烘焙Precomputed Realtime GI

烘焙时只存储场景内静态物体间的关系(?)，烘焙完成后可以自由调整光源或材质，并实时看到结果。
能有效降低实时光照计算的复杂度。
没有任何静态颜色被烘焙出来。
光源应选择Realtime类型。
Realtime Resolution值越高，间接光的效果越明显。

只烘焙Baked GI

传统的纯静态光照图烘焙，生成LightMaps；包含直接光源和反射的间接光源。
烘焙出的贴图在运行时无法改变，虽然可以叠加新贴图，但两者无法交互。
常用于低阶的手机开发。

  • Realtime灯光：不参与烘焙，同时作用于动态和静态物体
  • Baked灯光：仅作用于烘焙，不参与实时光照计算
  • Mixed灯光：参与烘焙，实时运行时仅作用于非静态物体

同时烘焙Static Lightmap与Realtime GI

光源的光照、物体间的二次反射都会直接烘焙在静态光照贴图上。
自发光及天空盒的辐射光线仍然可以实时变化并叠加作用于已烘焙的颜色上。
不建议使用这种方式渲染，同时消耗内存和计算资源，建议同时只使用一种。

一种做法

1.首先关闭Baked GI仅烘焙一些Precomputed Realtime GI
2.然后可以随意调整光源、材质、天空盒等并实时观察到场景的最终效果
3.最后关闭Realtime GI使用Baked GI进行光照贴图的烘焙

Unity-CharacterController

字数统计: 409

Unity中模拟物理碰撞效果有两套方法：
1.刚体和碰撞体(Rigidbody和Collider)
2.角色控制器(Character Controller)

在FPS类游戏中，角色在高速运动时可以立刻停下转身，并不符合物理规律，不适用Rigidbody，应使用Character Controller。
Character Controller允许在仅受制于碰撞影响的情况下控制物体的运动，不需要再添加collider和rigidbody.

Character Controller有以下特点：

  • 可以碰撞
  • 不受力，不能通过物理效果被其他对象影响
  • 不会对碰撞的物体施加力，通过OnControllerColliderHit() 自行编写其他物体的碰撞行为

public bool SimpleMove(Vector3 speed);
//以一定的速度移动角色控制器
//重力自动运用
//返回值表示是否着地

cc = GetComponet<CharacterController>();
cc.SimpleMove(speed * direction)

OnControllerColliderHit

void OnControllerColliderHit(ControllerColliderHit hit)

返回类型ControllerColliderHit，包含信息：

  • collider The collider that was hit by the controller.
  • controller The controller that hit the collider.
  • gameObject The game object that was hit by the controller.
  • moveDirection The direction the CharacterController was moving in when the collision occured.
  • moveLength How far the character has travelled until it hit the collider.
  • normal The normal of the surface we collided with in world space.
  • point The impact point in world space.
  • rigidbody The rigidbody that was hit by the controller.
  • transform The transform that was hit by the controller.

Variables

  • controller The controller that hit the collider.
  • gameObject The game object that was hit by the controller.

Unity-Basic2

发表于 2016-09-07
字数统计: 2,178

Game Narrative 故事叙述
Casual game 休闲游戏
Arcade game 街机游戏

Texture：

TextureType：贴图类型
AlphafromGrayscal: 使用灰度创建Alpha，当需要unity为贴图创建一条alpha通道，使用它转换得到的灰度值来渲染透明度
WrapMode：包裹模式，决定贴图会无限次平铺还是只会一次。Repeat就是重复平铺。Clamp会让贴图的边缘拉升并在改变大小时候填满任何缝隙
FilterMode：图像过滤模式，在拉伸时候对贴图过滤，比如[Point]点过滤，在很近看贴图时候，可能感觉不均。Bilinear它会时近看贴图显得模糊。Trilinear它会在不同mipmap层次之间模糊。
AnisoLevel：设置各项异性过滤器的级别，同属的可以理解为当你从一个比较陡的角度来看，贴图现实的可以有多细腻，渲染的等级越高，越占内存

Cast Shadows： Two Sided 两侧(如plane正反面)都能阻挡光照，产生阴影

ScaleFactor：缩放系数，使用这个属性在导入资源后设置其大小。
MeshCompression：网格压缩的比例[Off，Low，Medium，High]不可否认，压缩时可以节约内存的。
OptimizeMesh：为提高GPU性能记录网格顶点和索引。
GenerateColliders：是否在场景中让网格对象具有碰状体表现，通俗的说就是实心的，不可以穿体而过。
SwapUVs：交换UV，当你导入一个网格对象时候，着色器可能会使用错误的UV通道，若果发现资源异常[比如带光照图时候]，选中这个复选框。
Generate Lightmap UVs：为UV2生成光照贴图。

Normals&Tangents：该项设置计算法线切线
Normals：计算法线，import表示使用导入的网格法线，Calculate表示重新计算网格法线。
Tangents：计算切线，import表示使用导入的网格切线，Calulate表示重新计算切线。
Smoothing angle：法线平滑角度，告诉引擎在角度值为多少时候吧一个边缘视为硬边缘。
Split Tangents：是否分离切线,如果模型在场景渲染后现实UV缝隙，要开启它。

Materials:

Standard以及Standard(Specular setup)两者的区别是, Standard使用了金属模拟的外观, 而Standard(Specular Setup)使用了镜面模拟的外观。
使用中, 对于任何不是金属的材质, 都应该选择Standard(Specular Setup).

RenderingMode 渲染模式：
Opaque 不透明，例如石头，适用于所有的不透明的物体
Cutout 镂空，例如破布，透明度不是0%就是100%，不存在半透明的区域。
Fade 完全隐去，用于非现实的透明物体，与Transparent的区别为高光反射会随着透明度而消失。
Transparent 透明，例如玻璃，适用于像彩色玻璃一样的半透明物体，高光反射不会随透明而消失。

Albedo 就是Diffuse漫射的意思, 但是这里的Alpha通道还会控制透明程度，在无光照时显现的颜色
Normal 增强效果，使低精度贴图看起来像高精度
Occlution 在非直接光照如周围物体漫反射光照下的效果
Smoothness 物体表面光滑程度，控制光照在微观表面的反射效果
Specular 镜面反射，控制反射强度
Metallic 像金属一样的效果
Emission 发光度
Ambient Occlusion 环境光遮蔽，描绘物体和物体相交或靠近的时候遮挡周围漫反射光线的效果，可以解决或改善漏光、飘和阴影不实等问题

Tiling 表示UV坐标的缩放倍数
Offset 表示UV坐标的起始位置

RigidBody

Is Kinematic：是否开启动力学，开启后不在受物理引擎的影响，只能通过Transform属性来操作。此属性一般用来模拟平台的移动，或带有铰链关节链接刚体的动画。

Interpolate：插值，用于控制刚体运动抖动的情况，有以下三种值可选。
-None：没有插值
-Interpolate：内插值，基于前一帧的Transform来平滑此次的Transform。
-Extrapolate：外插值，基于下一帧的Transform来平滑此次的Transform。

Collsion Detection：碰撞检测，用于控制避免高速运动的对象穿过其他对象而未发生碰撞。
-Discrete：离散碰撞检测（默认值），与场景中其他多有碰撞体进行碰撞检测。
-Continuous：连续碰撞检测，此模式对物理性能会有很大影响，如果不需要对快速运动的对象进行碰撞检测，就是用离散模式。
-Continuous Dynamic：连续动态碰撞检测模式。

Physics.Raycast 光线投射,在场景中投下可与所有碰撞器碰撞的一条光线

Light：

Illumination 照明
Halo 光环
Flare 闪光，耀斑

Area Light,俗称面灯。没有任何实时光照的效果，仅用于Light baking即烘焙
Bounce Intensity:间接光照的效果强度
Culling mask 剔除遮罩，只照亮某些层
Cookie 投影遮罩。使用灰度纹理图准确的控制游戏灯光，做出格子光特效等
Soft Shadow: 相比Hard Shadow具有更加柔和的边缘

LightMap烘焙：
1.所要烘焙物体的mesh 必须要有合适的lightmapping uv。如果不确定的话，就在导入模型设置中勾选 Generate Lightmap UVs
2.任何Mesh Renderer, Skinned Mesh Renderer 或者 Terrain都要标注为static(lightmap static)

Light Probe Group：
LightProbe，可以让动态物体在烘焙好的场景里面受到光的照射效果，（墙烘焙，probe烘焙，球动态；灯光关闭后，墙不会反光照亮球，probe会照亮球）

Baking 灯光烘焙类型:
Baking 仅在烘焙时使用，运行时不可见；影响Baked LightMap，不会照亮Dynamic物体
RealTime 运行时使用；影响Realtime LightMap，照亮所有物体
Mixed 在运行和烘焙时都使用。
LightMap Bake + Light Probe 提供直接照射和间接照射的模拟方案，等同于RealTime的效果。运行时关闭RealTime，降低计算量
