统计基础

字数统计: 378

Average

  • Mode(众数): the most frequency
  • Median(中位数): sort and the number in the median
  • Mean(平均值): sum(numbers) / n

Variance

  • Absolute Deviation(绝对差): |xi - x`|
  • Variance(方差): sum(|xi - x`|^2) / n
  • Standard Deviation(标准差): root(var)
  • 样本的标准差: /n
  • 用样本估计总体的标准差: /(n-1)

协方差和相关系数

协方差(Covariance)是变量与均值之差乘积的期望:
$$ Cov(X,Y)=E\left [ (X-\mu_x)(Y-\mu_y)\right ] $$
两个变量同向变化则协方差为正；反向变化则协方差为负。

相关系数(Correlation Coefficient)是协方差除以变量各自的标准差:
$$ \rho = \frac{Cov(X,Y)}{\sigma_X\sigma_Y} $$
相关系数可以看做是剔除了变量量纲影响、标准化后的特殊协方差。
当相关系数为1时，说明两个变量变化的正向相似度最大，完全正相关；反之-1时为完全负相关。

正态分布

偏度(Skewness)是描述变量取值分布对称性的统计量，是样本的三阶标准化矩。
$$ Skewness = E\left[(\frac{x-\mu}{\sigma})^3\right] = \frac{E\left[(X-\mu)^3 \right]}{\sigma^3} $$

  • 等于0：标准正态分布
  • 大于0：右侧长尾，正偏
  • 小于0：左侧长尾，负偏

峰度(Kurtosis)是描述变量所有取值分布形态陡缓程度的统计量，是样本的四阶标准化矩。
$$ Kurtosis = E\left[(\frac{x-\mu}{\sigma})^4\right] = \frac{E\left[(X-\mu)^4 \right]}{\sigma^4}$$

  • 等于3：标准正态分布
  • 大于3：更陡峭，尖顶峰
  • 小于3：更平缓，平顶峰

Pandas is a module for data analysis based on Numpy.
There are two data types in pandas: Series and DataFrame.

Series

Series可以看做是定长的有序字典。
在数据运算时，Series可以自动对齐数据，即只对index相同的value运算，不重叠的索引处将引入NaN.

Series 的 index 和 values 实际仍为互相独立的 ndarray 数组。

DataFrame

DataFrame是一个表格型的数据结构。
它既有行索引也有列索引，values可以有多列，每列可以是不同的值类型，可以将各列看成是共享index的Series类型。
对齐操作将同时发生在行和列上。

Supervised Learning

Linear

  • Pros: Easy to interpret results, computationally inexpensive
  • Cons: Poorly models nonlinear data
  • Works with: Numeric values, Nominal values

k-Nearest Neighbors

  • Pros: High accuracy, insensitive to outliers, no assumptions about data(精度高，对异常值不敏感，无假设)
  • Cons: Computationally expensive, requires a lot of memory(计算消耗大)
  • Works with: Numeric values, nominal values

Decision Trees

  • Pros: Computationally cheap to use, easy for humans to understand learned results, missing values OK, can deal with irrelevant features
  • Cons: Prone to overfitting(容易过拟合)
  • Works with: Numeric values, nominal values

Naive Bayes

  • Pros: Works with a small amount of data, handles multiple classes
  • Cons: Sensitive to how the input data is prepared
  • Works with: Nominal values

Logistic Regression

  • Pros: Computationally inexpensive, easy to implement, knowledge representation easy to interpret(计算代价不高，易于实现和解释)
  • Cons: Prone to underfitting, may have low accuracy(容易欠拟合，不能保证精度)
  • Works with: Numeric values, nomial values

分类时，多重线性回归：z = w0x0 + w1x1 + w1x2 + … + wnxn，不能保证z ~ [0,1]。
故使用 f(z) = 1 / (1 + e^(-z)) ~ (0, 1) 映射至(0,1)
if f(z) > 0.5, then -> 1; else -> 0
只能分为2类，描述事件发生或不发生。

极大似然估计：选择w，使得L(w|x) = P(X=x)
求导数，解方程，使用Gradient Ascent(梯度上升)或者Gradient Descent(梯度下降)

Start with all weights set to 1
Repeat R number of times:
    Caculate the gradient of the entire dataset
    Update the weights vector by alpha*gradient
    Return the weights vector

Support Vector Machines

  • Pros: Low generalization error, computationally inexpensive, easy to interpret results
  • Cons: Sensitivie to tuning parameters and kernel choice, natively only handles binary classification
  • Works with: Numeric values, nominal values

The points closest to the separating hyperplane are known as support vectors.

AdaBoost

  • Pros: Low generalization error, easy to code, works with most classifiers, no parameters to adjust
  • Cons: Sensitive to outliers
  • Works with: Numeric values, nominal values

Unsupervised Learning

k-Means

  • Pros: Easy to implement
  • Cons: Can converge at local minima, slow on very large datasets
  • Works with: Numeric values

机器学习Tips

发表于 2016-05-21
字数统计: 395

问题分类

  • Supervised Learning: Approximation
    • Classification: y ~ {0,1}
    • Regression: y ~ [0,1] ~ R
  • Unsupervised Learning: Description
  • Reinforcement Learning

数据分类

原始数据可能包含多种不同的类别，每个类别的数据特征不同，比如宝可梦种类、交易数据的个人/商家等。
建模前应辨别是否需要将数据分离，针对不同特点的数据分别建模。

一般步骤

  1. 快速建模，容忍不精确
  2. 画出学习曲线，判断overfitting还是underfitting，是否需要更多的样本量/特征
  3. 引入的特征一般最高到二次，即$x_i^2$；因为我们默认对现实中的问题，平滑的曲线可能更正确(三次以上会有波动)
  4. 误差分析，从错误样本中找出改进方向

评估

  • Precision(准确率，查准率) = $\frac{TP}{TP + FP} = \frac{TP}{预测真}$ 所有预测为真的结果里，有多少是正确的
  • Recall(召回率，查全率) = $\frac{TP}{TP + FN} = \frac{TP}{实际真}$ 所有实际为真的样本里，有多少被预测了出来
  • F1 Score = $\frac{2PR}{P+R}$ 权衡precision和recall

Induction, Deduction, Abduction

Induction 归纳
Cause -> (Rule) -> Effect, not always true

Deduction演绎
Cause -> Rule -> (Effect), always true

Abduction溯因
(Cause) -> Rule -> Effect, not always true

With effect data, we use abduction to find its causes, then try to get a rule by induction, and finally deal with new data with deduction.

逻辑

  • 与(AND):$p\wedge q$，两个都为真{2}
  • 或(OR):$p\vee q$，至少一个为真{1,2}
  • 异或(XOR):$p\bigoplus q$，只有一个为真{1}
  • 或非(NOR = NOT + OR)，两个都为假{0}
  • 与非(NAND= NOT + AND)，至少一个假{0,1}

A CMMotionManager is the gateway to the motion services provided by iOS.

An app should create only a single instance of the CMMotionManager class.

Data Type

Accelerometer

accelerometerData.acceleration(x: Double, y: Double, z: Double)

Gyroscope

rotationRate(x: Double, y: Double, z: Double): raw data

Magnetometer

magnetometerData.magnetcField

DeviceMotion

  • attitude: CMAttitude, 朝向
  • rotationRate: bias removed
  • gravity
  • userAcceleration
  • mangeticField: device bias removed

Data Fetch

Pull: periodic sampling

Takes no arguments.
Recommended for games, for most games are interested only the lastest motion data.

Push

Takes an operation queue and a block handler for processing updates.

Must call stop when you no longer want your app to process updates.

An SKCameraNode is used to specify a position in the scene that the scene can be rendered from.
If a camera node is placed in the scene and the scene’s camera property is set to the camera node, then the scene is rendered using the camera node’s properties.

The scene is rendered so that the camera node’s origin is placed in the middle of the scene.

let cameraNode = SKCameraNode()
addChild(cameraNode)
camera = cameraNode
cameraNode.position = CGPoint(x: self.size.width / 2, y: self.size.height / 2)

Use camera node instead of moving background node manually.

The CGAffineTransform data structure represents a MATRIX used for affine transformations.
A transformation specifies how points in one coordinate system map to points in another coordinate system.

Creating

  • CGAffineTransformMakeScale(_ sx: CGFloat, _ sy: CGFloat) -> CGAffineTransform

Modification

  • CGAffineTransformTranslate(_ t: CGAffineTransform, _ tx: CGFloat, _ ty: CGFloat) -> CGAffineTransform: move the transform
  • CGAffineTransformScale
  • CGAffineTransformRotate
  • CGAffineTransformInvert(_ t: CGAffineTransform) -> CGAffineTransform
  • CGAffineTransformConcat(_ t1: CGAffineTransform, _ t2: CGAffineTransform) -> CGAffineTransform: combine two transform

Apply

  • CGRectApplyAffineTransform(_ rect: CGRect, _ t: CGAffineTransform) -> CGRect: apply the transform to rectangle

Evaluating
