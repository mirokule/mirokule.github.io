Unity-GI

发表于 2016-09-08
字数统计: 462

Unity中，实时照明(Realtime Lighting)的光线不会反射，只有投射光锥体范围内的对象表面才有光源影响，阴影是全黑的。
因此，引入Global Illumination全局照明，在直接光照的基础上，考虑了间接光照。

只烘焙Precomputed Realtime GI

烘焙时只存储场景内静态物体间的关系(?)，烘焙完成后可以自由调整光源或材质，并实时看到结果。
能有效降低实时光照计算的复杂度。
没有任何静态颜色被烘焙出来。
光源应选择Realtime类型。
Realtime Resolution值越高，间接光的效果越明显。

只烘焙Baked GI

传统的纯静态光照图烘焙，生成LightMaps；包含直接光源和反射的间接光源。
烘焙出的贴图在运行时无法改变，虽然可以叠加新贴图，但两者无法交互。
常用于低阶的手机开发。

  • Realtime灯光：不参与烘焙，同时作用于动态和静态物体
  • Baked灯光：仅作用于烘焙，不参与实时光照计算
  • Mixed灯光：参与烘焙，实时运行时仅作用于非静态物体

同时烘焙Static Lightmap与Realtime GI

光源的光照、物体间的二次反射都会直接烘焙在静态光照贴图上。
自发光及天空盒的辐射光线仍然可以实时变化并叠加作用于已烘焙的颜色上。
不建议使用这种方式渲染，同时消耗内存和计算资源，建议同时只使用一种。

一种做法

1.首先关闭Baked GI仅烘焙一些Precomputed Realtime GI
2.然后可以随意调整光源、材质、天空盒等并实时观察到场景的最终效果
3.最后关闭Realtime GI使用Baked GI进行光照贴图的烘焙

Unity-CharacterController

发表于 2016-09-08
字数统计: 409

Unity中模拟物理碰撞效果有两套方法：
1.刚体和碰撞体(Rigidbody和Collider)
2.角色控制器(Character Controller)

在FPS类游戏中，角色在高速运动时可以立刻停下转身，并不符合物理规律，不适用Rigidbody，应使用Character Controller。
Character Controller允许在仅受制于碰撞影响的情况下控制物体的运动，不需要再添加collider和rigidbody.

Character Controller有以下特点：

  • 可以碰撞
  • 不受力，不能通过物理效果被其他对象影响
  • 不会对碰撞的物体施加力，通过OnControllerColliderHit() 自行编写其他物体的碰撞行为

OnControllerColliderHit

void OnControllerColliderHit(ControllerColliderHit hit)

返回类型ControllerColliderHit，包含信息：

  • collider The collider that was hit by the controller.
  • controller The controller that hit the collider.
  • gameObject The game object that was hit by the controller.
  • moveDirection The direction the CharacterController was moving in when the collision occured.
  • moveLength How far the character has travelled until it hit the collider.
  • normal The normal of the surface we collided with in world space.
  • point The impact point in world space.
  • rigidbody The rigidbody that was hit by the controller.
  • transform The transform that was hit by the controller.

Game Narrative 故事叙述

Texture：

TextureType：贴图类型
AlphafromGrayscal: 使用灰度创建Alpha，当需要unity为贴图创建一条alpha通道，使用它转换得到的灰度值来渲染透明度
WrapMode：包裹模式，决定贴图会无限次平铺还是只会一次。Repeat就是重复平铺。Clamp会让贴图的边缘拉升并在改变大小时候填满任何缝隙
FilterMode：图像过滤模式，在拉伸时候对贴图过滤，比如[Point]点过滤，在很近看贴图时候，可能感觉不均。Bilinear它会时近看贴图显得模糊。Trilinear它会在不同mipmap层次之间模糊。
AnisoLevel：设置各项异性过滤器的级别，同属的可以理解为当你从一个比较陡的角度来看，贴图现实的可以有多细腻，渲染的等级越高，越占内存

Cast Shadows： Two Sided 两侧(如plane正反面)都能阻挡光照，产生阴影

ScaleFactor：缩放系数，使用这个属性在导入资源后设置其大小。
MeshCompression：网格压缩的比例[Off，Low，Medium，High]不可否认，压缩时可以节约内存的。
OptimizeMesh：为提高GPU性能记录网格顶点和索引。
GenerateColliders：是否在场景中让网格对象具有碰状体表现，通俗的说就是实心的，不可以穿体而过。
SwapUVs：交换UV，当你导入一个网格对象时候，着色器可能会使用错误的UV通道，若果发现资源异常[比如带光照图时候]，选中这个复选框。
Generate Lightmap UVs：为UV2生成光照贴图。

Normals&Tangents：该项设置计算法线切线
Normals：计算法线，import表示使用导入的网格法线，Calculate表示重新计算网格法线。
Tangents：计算切线，import表示使用导入的网格切线，Calulate表示重新计算切线。
Smoothing angle：法线平滑角度，告诉引擎在角度值为多少时候吧一个边缘视为硬边缘。
Split Tangents：是否分离切线,如果模型在场景渲染后现实UV缝隙，要开启它。

Materials:

Standard以及Standard(Specular setup)两者的区别是, Standard使用了金属模拟的外观, 而Standard(Specular Setup)使用了镜面模拟的外观。
使用中, 对于任何不是金属的材质, 都应该选择Standard(Specular Setup).

RenderingMode 渲染模式：
Opaque 不透明，例如石头，适用于所有的不透明的物体
Cutout 镂空，例如破布，透明度不是0%就是100%，不存在半透明的区域。
Fade 完全隐去，用于非现实的透明物体，与Transparent的区别为高光反射会随着透明度而消失。
Transparent 透明，例如玻璃，适用于像彩色玻璃一样的半透明物体，高光反射不会随透明而消失。

Albedo 就是Diffuse漫射的意思, 但是这里的Alpha通道还会控制透明程度，在无光照时显现的颜色
Normal 增强效果，使低精度贴图看起来像高精度
Occlution 在非直接光照如周围物体漫反射光照下的效果
Smoothness 物体表面光滑程度，控制光照在微观表面的反射效果
Specular 镜面反射，控制反射强度
Metallic 像金属一样的效果
Emission 发光度
Ambient Occlusion 环境光遮蔽，描绘物体和物体相交或靠近的时候遮挡周围漫反射光线的效果，可以解决或改善漏光、飘和阴影不实等问题

Tiling 表示UV坐标的缩放倍数
Offset 表示UV坐标的起始位置

RigidBody

Is Kinematic：是否开启动力学，开启后不在受物理引擎的影响，只能通过Transform属性来操作。此属性一般用来模拟平台的移动，或带有铰链关节链接刚体的动画。

Interpolate：插值，用于控制刚体运动抖动的情况，有以下三种值可选。
-None：没有插值
-Interpolate：内插值，基于前一帧的Transform来平滑此次的Transform。
-Extrapolate：外插值，基于下一帧的Transform来平滑此次的Transform。

Collsion Detection：碰撞检测，用于控制避免高速运动的对象穿过其他对象而未发生碰撞。
-Discrete：离散碰撞检测（默认值），与场景中其他多有碰撞体进行碰撞检测。
-Continuous：连续碰撞检测，此模式对物理性能会有很大影响，如果不需要对快速运动的对象进行碰撞检测，就是用离散模式。
-Continuous Dynamic：连续动态碰撞检测模式。

Physics.Raycast 光线投射,在场景中投下可与所有碰撞器碰撞的一条光线

Light：

Illumination 照明
Halo 光环
Flare 闪光，耀斑

Area Light,俗称面灯。没有任何实时光照的效果，仅用于Light baking即烘焙
Bounce Intensity:间接光照的效果强度
Culling mask 剔除遮罩，只照亮某些层
Cookie 投影遮罩。使用灰度纹理图准确的控制游戏灯光，做出格子光特效等
Soft Shadow: 相比Hard Shadow具有更加柔和的边缘

LightMap烘焙：
1.所要烘焙物体的mesh 必须要有合适的lightmapping uv。如果不确定的话，就在导入模型设置中勾选 Generate Lightmap UVs
2.任何Mesh Renderer, Skinned Mesh Renderer 或者 Terrain都要标注为static(lightmap static)

Light Probe Group：
LightProbe，可以让动态物体在烘焙好的场景里面受到光的照射效果，（墙烘焙，probe烘焙，球动态；灯光关闭后，墙不会反光照亮球，probe会照亮球）

Baking 灯光烘焙类型:
Baking 仅在烘焙时使用，运行时不可见；影响Baked LightMap，不会照亮Dynamic物体
RealTime 运行时使用；影响Realtime LightMap，照亮所有物体
Mixed 在运行和烘焙时都使用。
LightMap Bake + Light Probe 提供直接照射和间接照射的模拟方案，等同于RealTime的效果。运行时关闭RealTime，降低计算量

Audio：

Gapless looping : 无缝循环，对于loop音频，有可能由于压缩导致破坏循环，这种情况下可以通过设置这个变量来解决

Input类是处理输入的接口。

Key与物理按键对应，如键盘、鼠标、摇杆，通过名称或编码KeyCode访问。
Button与虚拟按键对应，如“Fire 1”，通过名称访问。

鼠标

  • mousePosition: 鼠标位置，Vector3(x, y, 0)

  • GetMouseButtonDown(n): 鼠标按下的第一帧返回true，0左键，1右键，2中键

  • GetMouseButtonUp(n): 鼠标松开的第一帧返回true

  • GetMouseButton(n): 鼠标按下期间持续返回true

  • GetAxis(“Mouse X”): 得到一帧内鼠标在水平方向的移动距离，有平滑如0.1, 0.34, 0.78

  • GetAxis(“Mouse Y”): 得到一帧内鼠标在垂直方向的移动距离
  • GetAxisRaw(): 不使用平滑器，只返回-1,0,1，用来控制角色移动时没有加速和刹车过程

    键盘

  • GetKey
  • GetKeyDown(Keycode key);
  • GetKeyUp
  • GetButtonDown(string buttonName);
  • GetAxis(“Horizontal”)
  • GetAxis(“Vertical”)

触摸

  • GetTouch
  • Touches 当前所有触摸状态列表
  • touchCount 触摸的点数

返回的是数据保存在Touch结构体中，Touch结构体的变量如下：

  • fingerid 唯一索引
  • position
  • deltaPosition
  • deltaTime
  • tapCount
  • phase 状态描述

Phase的状态如表

  • Began 手指刚触碰屏幕
  • Moved 在屏幕上移动
  • Stationary 自上一帧没有移动
  • Ended 离开屏幕
  • Canceled 系统取消了跟踪触摸

每个GameObject都有一个Transform，记录了对象的位置、大小、旋转等信息。

成员变量：

  • position: 世界坐标系中的位置
  • eulerAngles: 世界坐标系中，以欧拉角表示的旋转

  • rotation: 世界坐标中，以四元数表示的旋转

  • localPosition: 父对象局部坐标系中的位置

  • localEulerAngles: 父对象局部坐标系中的欧拉角

  • localRotation: 父对象局部坐标系中四元数表示的旋转

  • up: 对象在世界坐标系中的上方向, y轴？

  • right: 对象在世界坐标系中的右方向, x轴？

  • forward: 对象在世界坐标系中的前方向, z轴？

  • parent: 父对象的transform组件

  • childCount: 子对象的数量

成员函数：

  • Translate: 按指定的方向和距离平移
  • Rotate: 按指定的欧拉角旋转
  • RotateAround: 按指定的轴和角度旋转

  • LookAt: 旋转使得自身的前方向(forward)指向目标位置

  • TransformDirection: 将一个方向从局部坐标系变换到世界坐标系

  • InverseTransformDirection: 反向
  • TransformPoint: 将一个位置从局部坐标系变换到世界坐标系

  • InverseTransformPoint: 反向

  • DetachChildren: 与所有子物体解除父子关系

  • Find: 按名称查找子对象
  • IsChildOf: 判断是否为指定对象的子对象

协程(Coroutine)不是线程(Thread)。
协程和线程的区别是：协程避免了无意义的调度，由程序员自己承担；由此可以提高性能，但也失去了标准线程使用多CPU的能力。
Unity中所有的脚本和代码都在一个主线程里运行，通过每帧检测的方式，在函数间来回切换。
协程控制的是函数是否在当前帧内执行。

  • yield retrun null: 当前帧执行到此结束，后面的语句在下一帧执行
  • yield return new WaitForSeconds(n): 后面的语句在等待n秒后执行

Coroutine

  1. Return type of function should be “IEnumerator”
  2. Using “yield return”：将程序挂起
  3. Using “StartCoroutine(function())”
开始协程

StartCoroutine(IEnumerator routine);

停止协程

StopCoroutine(routine);
StopAllCoroutine();

Invoke

Invoke(“Function”, n)：在n秒后执行函数Function

  1. 应该在脚本的生命周期(Start, Update, FixedUpdate, LateUpdate)中被调用;
  2. 只能接受没有参数的方法
  3. 受Time.ScaleTime影响；当Time.ScaleTime=0时，Invoke()永远不会执行function

Rotation

旋转本质上需要知道 1个向量 + 1个角度，共4个值。有3种表示方法：

  • Matrix(矩阵): 使用 4*4 矩阵，共16个值，计算时有额外的时间和空间消耗；

  • Euler(欧拉): 使用3个值(xRotation, yRotation, zRotation)，是一系列坐标轴旋转的组合，按顺序沿各个轴旋转；会带来万向节锁问题。
    Gimbal Lock(万向节锁)：当欧拉旋转的第二次旋转角度为90°时，第三个轴就被转到了第一个轴相同的方向，因此丢失了一个自由度。

  • Quaternion(四元数): 本质上是一个高阶复数，即 x = a + bi + cj + dk，效率更高

    rigidBody.rotation = Quaternion.Euler(0.0f, 0.0f, rigidBody.velocity.x * -1);

乘法

  1. quaternion quaternio，例如q = q1 q2，表示将一个点先q1旋转，再q2旋转
  2. quaternion Vector3，例如p = q v，表示将向量v进行q旋转，得到旋转后的向量p.
    Quaternion必须位于乘号的左边。

Average

  • Mode(众数): the most frequency
  • Median(中位数): sort and the number in the median
  • Mean(平均值): sum(numbers) / n

Variance

  • Absolute Deviation(绝对差): |xi - x`|
  • Variance(方差): sum(|xi - x`|^2) / n
  • Standard Deviation(标准差): root(var)
  • 样本的标准差: /n
  • 用样本估计总体的标准差: /(n-1)

协方差和相关系数

协方差(Covariance)是变量与均值之差乘积的期望:
$$ Cov(X,Y)=E\left [ (X-\mu_x)(Y-\mu_y)\right ] $$
两个变量同向变化则协方差为正；反向变化则协方差为负。

相关系数(Correlation Coefficient)是协方差除以变量各自的标准差:
$$ \rho = \frac{Cov(X,Y)}{\sigma_X\sigma_Y} $$
相关系数可以看做是剔除了变量量纲影响、标准化后的特殊协方差。
当相关系数为1时，说明两个变量变化的正向相似度最大，完全正相关；反之-1时为完全负相关。

正态分布

偏度(Skewness)是描述变量取值分布对称性的统计量，是样本的三阶标准化矩。
$$ Skewness = E\left[(\frac{x-\mu}{\sigma})^3\right] = \frac{E\left[(X-\mu)^3 \right]}{\sigma^3} $$

  • 等于0：标准正态分布
  • 大于0：右侧长尾，正偏
  • 小于0：左侧长尾，负偏

峰度(Kurtosis)是描述变量所有取值分布形态陡缓程度的统计量，是样本的四阶标准化矩。
$$ Kurtosis = E\left[(\frac{x-\mu}{\sigma})^4\right] = \frac{E\left[(X-\mu)^4 \right]}{\sigma^4}$$

  • 等于3：标准正态分布
  • 大于3：更陡峭，尖顶峰
  • 小于3：更平缓，平顶峰

  • Works with: Nominal values

  • Works with: Numeric values, nomial values

机器学习Tips

发表于 2016-05-21
字数统计: 395

问题分类

  • Supervised Learning: Approximation
    • Classification: y ~ {0,1}
    • Regression: y ~ [0,1] ~ R
  • Unsupervised Learning: Description
  • Reinforcement Learning

数据分类

原始数据可能包含多种不同的类别，每个类别的数据特征不同，比如宝可梦种类、交易数据的个人/商家等。
建模前应辨别是否需要将数据分离，针对不同特点的数据分别建模。

一般步骤

  1. 快速建模，容忍不精确
  2. 画出学习曲线，判断overfitting还是underfitting，是否需要更多的样本量/特征
  3. 引入的特征一般最高到二次，即$x_i^2$；因为我们默认对现实中的问题，平滑的曲线可能更正确(三次以上会有波动)
  4. 误差分析，从错误样本中找出改进方向

评估

  • Precision(准确率，查准率) = $\frac{TP}{TP + FP} = \frac{TP}{预测真}$ 所有预测为真的结果里，有多少是正确的
  • Recall(召回率，查全率) = $\frac{TP}{TP + FN} = \frac{TP}{实际真}$ 所有实际为真的样本里，有多少被预测了出来
  • F1 Score = $\frac{2PR}{P+R}$ 权衡precision和recall

Induction, Deduction, Abduction

Induction 归纳
Cause -> (Rule) -> Effect, not always true

Deduction演绎
Cause -> Rule -> (Effect), always true

Abduction溯因
(Cause) -> Rule -> Effect, not always true

With effect data, we use abduction to find its causes, then try to get a rule by induction, and finally deal with new data with deduction.

逻辑

  • 与(AND):$p\wedge q$，两个都为真{2}
  • 或(OR):$p\vee q$，至少一个为真{1,2}
  • 异或(XOR):$p\bigoplus q$，只有一个为真{1}
  • 或非(NOR = NOT + OR)，两个都为假{0}
  • 与非(NAND= NOT + AND)，至少一个假{0,1}
