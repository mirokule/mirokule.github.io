Anchors

所有UI对象都有锚点(anchor)，锚点组成一个矩形，坐标归一化，代表“在父对象中的相对位置”。

Unity-SphereCast

发表于 2016-11-23
字数统计: 227

Casts a sphere along a ray and returns detailed information on what was hit.

This is useful when a Raycast does not give enough precision, because you want to find out if an object of a specific size, such as a character, will be able to move somewhere without colliding with anything on the way. Think of the sphere cast like a thick raycast. In this case the ray is specified by a start vector and a direction.

Returns true when the sphere intersects any collider.

GUI

UI交互的一般步骤

(1)在场景中创建UI控件(按钮、文本、控制器)
(2)编写控制器脚本，实现操作UI时的功能
(3)将脚本附加到控制器上
(4)将相关UI控件连接到脚本上,或者将UI控件的event trigger连接到脚本函数上

其他

1.event trigger连接带参数的脚本函数时，要在面板顶部的Dynamic XXX部分选择该函数，而不是在底部的Static Parameters部分。
在Static Parameters中选择时只会发送提前定义的单个字符串，而在Dynamic中选择发送的是输入域中的任何内容。

1.如果没有标明成员函数或成员变量的访问权限级别，
C++中，struct中默认的是public，class中默认的是private
C#中，struct和class中都是private的。

2.struct定义的是值类型，值类型的实例在栈上分配内存，由编译器自动分配和释放
class定义的是引用类型，引用类型的实例在堆上分配内存，由用户分配和释放

3.C#可以定义Finalizer和Dispose方法来释放资源，类似于C++的析构函数，但运行时机不确定
静态构造函数，在类型第一次被使用前自动调用，且保证只调用一次

4.C++支持多继承，C#类只能继承一个基类中的实现但可以实现多个接口。

5.C# 中没有全局方法和全局变量：方法和变量必须包含在类型声明（如 class 或 struct）中。

6.C# 不能重写非虚方法或静态方法。能重写的基方法必须是 virtual、abstract的。

  • virtual：基类必须有实现，子类可以重写或不重写，用于“可以重写的方法”
  • abstract：基类必须没有实现，子类必须有实现，类似接口，用于“必须重写的方法”
  • override：子类重写的方法，前缀必须加上override

7.接口是抽象的集合，不能存在实现的功能
抽象类不能被实例化，存在抽象方法，具有类的其他特性；可以有已经实现的方法

8.private类型完全私有，不能被子类继承，只有当前类中的成员能访问；
protected类型受保护，能被子类继承，只有当前类的成员、子类成员能访问；
public类型完全公开，能被子类继承，大家都可以访问。

AI-PathFinding

发表于 2016-11-23
字数统计: 905

地图划分

1.Grid (方格)

将搜索区域划分成了方形网格。
没有障碍物的地方画上方格，选取方格中心作为可以行走的点。

优点：划分简单，容易结合寻路算法使用。
缺点：移动是不连续的；网格数较多时需要资源计算和存储。

2.Navigation Mesh(导航网格)

利用了凸多边形的性质：在凸多边形上的一个点走到另一个点，路径一定在多边形内部。

生成步骤：
（1）通过地图的边界点和障碍物的边界点，将地图划分成许多三角形；
（2）将相邻的三角形合并形成凸多边形；
（3）将凸多边形继续不断合并为凸多边形；
（4）将凸多边形每条边的中点放置路径点，将每个多边形的路径点连线。

优点：降低搜寻计算量，移动更加自然。
缺点：实现复杂。

Dot 点积

u.v = |u||v|cos<u,v> = ux * vx + uy * vy，得到标量

在计算机图形学和游戏编程中，点积的重要用途之一是计算向量在给定方向上的分量(投影)。

Proj u = (u.v) * v / (|v|*|v|)

另外用于计算夹角
点积大于0则两个向量的夹角小于90度,锐角; 点积等于0，直角; 点积小于0,钝角

Vector3 a = new Vector3(1,1,1);
Vector3 b = enw Vector3(1,5,1);

float result = Vector3.Dot(a,b);
float angle = Vector3.Angle(a,b); //夹角，返回[0 - 180]

Cross 叉积

仅当向量包含3个或更多分量时，叉积才有意义。

u*v = |u|*|v|*sin<u,v> * n

其中n是uv平面的单位法向量。
实际应用中，使用矩形形式计算单位法向量n，很少直接使用叉积。

// c = a x b，得到向量，向量的模为 |c| = |a||b|sin<a,b>，且向量c垂直于a、b组成的平面，a x b = - b x a(左/右手定则?)
Vector3 c = Vector3.Cross(a,b);

// 判断顺时针、逆时针方向，指定XZ平面，考察叉积的Y轴分量
if (c.y > 0)
{ ... }

// 返回夹角，给angle标明方向
float sign = Mathf.Sign(Vector3.Dot(c.normalized, Vector3.Cross(a.normalized, b.normalized)));
float signed_angle = angle * sign;
