sklearn-CrossValidation

发表于 2018-03-03
字数统计: 819

如果模型验证和训练使用同一批数据，可能会过拟合，故应将训练集划分。

  • 单一划分：train_test_split，会使训练样本数减少
  • 交叉验证：k-fold等，将数据划分成k份，每次留一份验证，循环k次取平均值

交叉验证要素

  • 划分方式：划分几份，KFold还是Shuffle
  • 评价策略：score选择

单一划分

1. train_test_split

sklearn.model_selection.train_test_split(arrays, *options)
随机划分训练集和测试集。

参数：

  • *arrays: 输入可以是list, ndarray, dataframe或者scipy-sparse matrices
  • test_size: 默认0.25，(0.0~1.0)表示比例，整数表示样本数
  • train_size: 默认None，(0.0~1.0)表示比例，整数表示样本数
  • random_state: 随机种子
  • shuffle: 默认True，在划分前是否要Shuffle
  • stratify: 默认None，若要分层则传入类标签

返回：

  • splitting: 划分后的list

2. cross_val_score

sklearn.model_selection.cross_val_score(estimator, X, y=None, groups=None, scoring=None, cv=None, n_jobs=1, verbose=0, fit_params=None, pre_dispatch=’2*n_jobs’)

  • 划分方式：cv为整数，默认KFold或StratifiedKFold划分;也可以令cv=其他划分方式
  • 评价策略：默认使用模型的score方法，也可以用scoring自定义

参数：

  • estimator: 模型
  • X: 输入特征iris.data
  • y: 预测目标iris.target
  • groups: 划分时的类标签
  • scoring: 评分函数/字符串
  • cv: 折叠次数
  • n_jobs: 使用的CPU数
  • verbose: 冗余日志输出？
  • fit_params: 传递给fit的参数
  • pre_cispatch: 并发任务数

返回：

  • scores: 得分数组，长度等于划分次数

3. cross_validate

sklearn.model_selection.cross_validate(estimator, X, y=None, groups=None, cv=None, n_jobs=1, verbose=0, fit_params=None, pre_dispatch=’2*n_jobs’, return_train_score=’warn’)
和cross_val_score有两点不同：

  • 评分策略：允许在evaluation中使用多种metrics
  • 更多信息：返回一个包含更多评分信息的字典，而不单单是scores数组

参数：

  • estimator, X, y, groups, cv, n_jobs, verbose, fit_params, pre_dispatch: 同上
  • scoring：支持同时返回多个metrics
  • return_train_score: 默认’warn’ = True + a warning，0.21中将默认False

返回：

  • scores: 字典，键值包括{‘test_score’, ‘train_score’, ‘fit_time’, ‘score_time’}

sklearn.Preprocessing

名词辨析

  • Standardization: 标准化，根据中心极限定理，将数据转变为近似服从标准正态分布N(0,1)
  • Normalization: 规范化，将数据映射到相同的范围，最常见的是[0,1]，此时也叫归一化，消除量纲影响，对GDA友好
  • Regularization: 正则化，在模型的cost function里增加参数惩罚项，避免过拟合

1. StandardScaler, Scale

标准化，将特征减去均值，再除以标准差，从而服从标准正态分布N(0,1)
默认列向量，每行是一次采样。

参数：

  • copy: 为True则返回拷贝，为False则修改原数据，不一定总是有效
  • with_mean: 为True则在缩放前先减去均值
  • with_std: 为True则缩放到单位方差
  • axis: 默认为0缩放列向量(每个特征)，为1则缩放行向量(每次采样)

属性：

  • scale_:
  • mean_: 特征原均值
  • var_: 特征原方差
  • n_samples_seen_: 处理的样本数

方法：

  • fit(X, y=None): 计算均值和标准差
  • transform(X, copy=None): 变换数据
  • fit_transform(X, y=None, **fit_params): 计算，并变换数据
  • inverse_transform(X, copy=None): 逆向变换，得到原数据
  • partial_fit(X, y=None): 在线计算均值和标准差
  • get_params(deep=True): 返回参数
  • set_params(**params): 设定参数

2. Normalizer, Normalize

默认列向量，每行是一次采样。
将每行(所有变量的一次采样)缩放到单位长度，l1是曼哈顿距离，l2是欧几里德距离。
比如某行为(a,b,c)，l2 norm，则缩放后$a^2 + b^2 + c^2 = 1$

参数：

  • norm: 可以选’l1’, ‘l2’, ‘max’
  • copy: 为False则修改原数据，为True则返回拷贝
  • axis: 为0则缩放列向量(每个特征), 默认为1缩放行向量(每次采样)

方法：

  • fit(X, y=None): 无作用，只是为了实现接口
  • transform(X, copy=None): 缩放每个非零行
  • fit_transform(X, y=None, **fit_params): 带参数缩放
  • set_params(**params): 设定参数
  • get_params(deep=True): 返回参数

魔术命令

魔术命令(Magic Command)是以百分号(%)为前缀的命令，为常见任务提供便利。
使用问号(?)同样可以查看命令选项。

  • %run 可以执行一个py脚本
  • %time 执行一条语句，报告执行时间
  • %timeit 多次执行同一语句，报告执行时间
  • %debug 进入调试器
  • %pdb 在异常发生后自动进入调试器
  • %paste 执行剪贴板中的代码
  • %cpaste 打开一个特殊提示符以便手工粘贴待执行的Python代码
  • %hist 打印命令的输入历史
  • %who、%whos、%who_ls 显示命名空间中定义的变量
  • %reset 删除命名空间中的全部变量
  • %quickref 显示IPython的快速参考
  • %magic 显示所有魔术命令的详细文档

IPython的另一个重要特点是跟操作系统shell结合得非常紧密，可以在IPython中直接使用Linux系统命令，而不用像Python那样import os。调用shell命令的魔术命令：

  • %pwd 返回当前工作目录
  • %cd directory 更改当前工作目录
  • %pushd directory 将当前目录入栈，并转向目标目录
  • %popd 弹出栈顶目录，并转向该目录
  • %alias alias_name cmd 为系统shell命令定义别名
  • !cmd 在系统shell中执行cmd
